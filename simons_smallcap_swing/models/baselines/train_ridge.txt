\begin{abstract}
Este documento analiza el módulo \texttt{models/baselines/train\_ridge.py}, que entrena una \textbf{regresión lineal regularizada L2 (Ridge)} para targets continuos como referencia estable y explicable frente a modelos más complejos (GBDT, etc.) en un sistema de trading cuantitativo. Se discute la formulación de Ridge, la selección de \texttt{alpha} mediante \textbf{validación cruzada temporal purged}, las métricas de calidad OOF/OOS y cómo este baseline contribuye a un ecosistema de modelos de nivel industrial, conectando con estado del arte académico (Ridge, CV temporal, multicolinealidad).
\end{abstract}

\section{Objetivo y contexto del módulo}

El script \texttt{train\_ridge.py} pertenece al subpaquete \texttt{models/baselines} del repositorio \texttt{JimSimons}. Su objetivo principal es:

\begin{itemize}
  \item Entrenar un modelo de \textbf{regresión lineal regularizada L2 (Ridge)} para targets continuos (por ejemplo, retornos raw, retornos normalizados, señales continuas).
  \item Proporcionar una \textbf{referencia interpretable y estable} sobre la que comparar modelos no lineales (GBDT, XGBoost, redes neuronales).
  \item Establecer un \textbf{baseline de regresión lineal} útil para:
    \begin{itemize}
      \item Evaluación de ganancia incremental de modelos complejos.
      \item Análisis de importancia de variables (a través de coeficientes \( \beta_j \)).
    \end{itemize}
\end{itemize}

En el contexto de swing trading, Ridge actúa como \textbf{baseline de regresión continua} con buena robustez frente a multicolinealidad y una explicación directa de la relación lineal entre features y target. [web:68][web:71]

\section{Formulación matemática de Ridge}

Sea \( X \in \mathbb{R}^{n \times p} \) la matriz de features, \( y \in \mathbb{R}^n \) el target continuo y \( \beta \in \mathbb{R}^p \) el vector de coeficientes. El estimador de Ridge se define como:

\[
  \hat{\beta}_{\text{Ridge}}(\alpha)
  = \arg\min_{\beta} \left\{
    \| y - X\beta \|^2_2
    + \alpha \|\beta\|_2^2
  \right\},
\]

donde:
\begin{itemize}
  \item \( \|\beta\|_2^2 = \sum_{j=1}^p \beta_j^2 \) es la penalización L2.
  \item \( \alpha \ge 0 \) es el parámetro de regularización (“shrinkage parameter”).
\end{itemize}

La solución cerrada de Ridge viene dada por:

\[
  \hat{\beta}_{\text{Ridge}}(\alpha) = (X^\top X + \alpha I)^{-1} X^\top y.
\]

Cuando \( \alpha \to 0 \), el estimador se aproxima al de mínimos cuadrados ordinarios (OLS); cuando \( \alpha \to \infty \), todos los coeficientes se contraen hacia 0. [web:68][web:67]

\subsection{Ventajas de Ridge frente a OLS}

\begin{itemize}
  \item \textbf{Mejora de condicionamiento}: Ridge “stabiliza” el sistema cuando \( X^\top X \) está mal condicionada (alta multicolinealidad).
  \item \textbf{Trade‑off sesgo/varianza}: introduce un pequeño \emph{bias} a cambio de una reducción importante de la varianza de los coeficientes.
  \item \textbf{Comportamiento predecible}: la solución de Ridge es única y continua en \( \alpha \) y adecuada para CV temporal y tracking de coeficientes por fold. [web:68]
\end{itemize}

\section{Alcance funcional del módulo}

\subsection{Entrenamiento temporal purged}

El módulo emplea \textbf{CV temporal purged} para evitar \emph{data leakage} en targets construidos a partir de ventanas temporales:

\begin{itemize}
  \item Se cargan \texttt{features}, \texttt{labels} y \texttt{labels/splits/purged\_splits\_latest.parquet}.
  \item Cada fold mantiene el orden temporal y excluye obervaciones contaminadas por el rango de test.
  \item Opcionalmente se puede introducir \textbf{embargo} tras el test para evitar interferencias de efecto de ventana. [web:66][web:57]
\end{itemize}

Esto asegura que el modelo nunca “ve” future leakage en el label ni en el feature.

\subsection{Selección/calibración de \(\alpha\)}

\begin{itemize}
  \item El módulo permite una \texttt{alpha\_grid} (p. ej. \( [10^{-6}, 10^{-4}, 10^{-2}, 1] \)) o un \texttt{alpha\_fixed}.
  \item Mediante \texttt{RidgeCV} o \texttt{GridSearchCV} sobre \texttt{alpha} se selecciona el valor óptimo por fold o globalmente, minimizando RMSE/MSE en validación temporal.
\end{itemize}

La literatura muestra que la selección de \( \alpha \) por CV temporal contribuye a un buen \textbf{trade‑off entre overfitting y underfitting}, incluso en contextos de alto número de variables. [web:69][web:75]

\section{Entradas y salidas}

\subsection{Entradas}

El módulo recibe:
\begin{itemize}
  \item \texttt{features\_path}: ruta a features preprocesadas (\texttt{features/store}).
  \item \texttt{labels\_path}: ruta a targets continuos (\texttt{labels/store}).
  \item \texttt{splits\_path}: splits temporales \textbf{purged} (\texttt{labels/splits/purged\_splits\_latest.parquet}).
  \item \texttt{config\_path}: configuración YAML (\texttt{configs/models/ridge.yaml}).
  \item \texttt{run\_id}: identificador de la corrida (p. ej. \texttt{train\_ridge\_20260305\_001}).
\end{itemize}

\subsection{Salidas}

Por cada \texttt{run\_id}:

\begin{itemize}
  \item Modelos por fold:
    \texttt{models/artifacts/baselines/ridge/{run\_id}/fold\_{k}/model.pkl}
  \item Predicciones OOF:
    \texttt{models/predictions/baselines/ridge/{run\_id}/oof.parquet}
  \item Informes de métricas:
    \texttt{models/reports/baselines/ridge/{run\_id}/metrics\_summary.json}
  \item Manifesto de ejecución:
    \texttt{models/reports/baselines/ridge/{run\_id}/manifest.json}
\end{itemize}

\section{Parámetros clave y configuración}

\begin{itemize}
  \item \texttt{alpha\_grid} o \texttt{alpha\_fixed}: valores de regularización explorados.
  \item \texttt{fit\_intercept}: si se ajusta el intercepto \( \beta_0 \) o se asume que \( X \) está centrado.
  \item \texttt{standardize\_features}: si se aplica \texttt{StandardScaler} a \( X \) antes de entrenar.
  \item \texttt{sample\_weight\_mode}: modo de ponderación de muestras (por ejemplo, por volumen o por volatilidad).
  \item \texttt{target\_column}: columna específica del dataset de labels que se usa como \( y \).
\end{itemize}

El uso de \texttt{standardize\_features=True} hace que los coeficientes sean comparables y robustos frente a escalas distintas de las variables. [web:70][web:79]

\section{Reglas de PIT y buenas prácticas}

\begin{itemize}
  \item \textbf{Entrenamiento solo en train folds purged}: ningún dato de valid/test es usado para estimar \( \beta \).
  \item \textbf{Valid/test fuera del ajuste de hiperparámetros}: el conjunto de test final se mantiene oculto para la selección de \texttt{alpha}.
  \item \textbf{Sin leakage entre fechas}: se respeta la estructura temporal de etiquetas (p. ej. retornos, eventos de mercado).
\end{itemize}

Esto garantiza que las métricas OOF/OOS son una estimación honesta del rendimiento futuro en datos nuevos. [web:51][web:66]

\section{Flujo de ejecución}

\begin{enumerate}
  \item \textbf{Cargar datos y alinear por \((date, symbol)\)}:
    \begin{itemize}
      \item Se cargan \texttt{features} y \texttt{labels} y se alinean por índice temporal y símbolo.
      \item Se verifica que cada fecha/símbolo tiene su correspondiente label.
    \end{itemize}
  \item \textbf{Preparar matriz \( X,y \) por fold}:
    \begin{itemize}
      \item Para cada fold, se crea un subconjunto \( X_{\text{train}}, y_{\text{train}} \) y \( X_{\text{test}}, y_{\text{test}} \).
    \end{itemize}
  \item \textbf{Escalado de features}:
    \begin{itemize}
      \item Si \texttt{standardize\_features=True}, se aplica \texttt{StandardScaler} ajustado solo sobre el \texttt{train} de cada fold.
    \end{itemize}
  \item \textbf{Entrenar Ridge y evaluar OOF/OOS}:
    \begin{itemize}
      \item Se entrena \texttt{Ridge} o \texttt{RidgeCV} sobre \texttt{alpha\_grid}.
      \item Se guardan coeficientes, predicciones y métricas.
    \end{itemize}
  \item \textbf{Persistir modelos y reportes}:
    \begin{itemize}
      \item \texttt{model.pkl}, \texttt{oof.parquet}, \texttt{metrics\_summary.json} y \texttt{manifest.json}.
    \end{itemize}
\end{enumerate}

\section{Métricas recomendadas}

\begin{itemize}
  \item \textbf{RMSE}:
    \[
      \text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^n (y_i - \hat{y}_i)^2}
    \]
  \item \textbf{MAE}:
    \[
      \text{MAE} = \frac{1}{n} \sum_{i=1}^n |y_i - \hat{y}_i|
    \]
  \item \textbf{Correlación} entre \texttt{y} e \texttt{pred}:
    \[
      \rho(y, \hat{y}) = \text{Corr}(y, \hat{y})
    \]
  \item \textbf{R²} (explicación de varianza):
    \[
      R^2 = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}
    \]
\end{itemize}

Estas métricas permiten comparar Ridge con modelos más complejos y evaluar cuánto valor añadido aporta la no linealidad frente a una referencia lineal simple. [web:68][web:71]

\section{Validaciones y calidad del pipeline}

\begin{itemize}
  \item \textbf{Columnas requeridas presentes}:
    \begin{itemize}
      \item Verificar que existen \texttt{date, symbol, target\_column} y todas las features necesarias.
    \end{itemize}
  \item \textbf{Sin NaN/inf tras preprocessing}:
    \begin{itemize}
      \item Lanzar error si aparecen \texttt{NaN} o \texttt{inf} en \texttt{X} o \texttt{y}.
    \end{itemize}
  \item \textbf{Métricas por fold completas}:
    \begin{itemize}
      \item Que cada fold tenga métricas asociadas (RMSE, MAE, R², etc.).
      \item Sin folds “vacíos” ni divisiones temporales corruptas.
    \end{itemize}
\end{itemize}

\section{Estado del arte académico e industrial}

\subsection{Ridge regression y finanzas cuantitativas}

\begin{itemize}
  \item Ridge es ampliamente utilizado como método de regresión lineal regularizada para:
    \begin{itemize}
      \item Predicción de retornos continuos.
      \item Modelos de rendimiento de carteras y factores.
    \end{itemize}
    [web:68][web:69]
  \item En contextos de trading, se usa como base para:
    \begin{itemize}
      \item \textbf{Baselines lineales} para sistemas de momentum y mean‑reversion.
      \item \textbf{Rolling Ridge} con ventanas móviles que se ajustan a la dinámica temporal del mercado. [web:71][web:74]
    \end{itemize}
\end{itemize}

\subsection{Regresión lineal regularizada y CV temporal}

\begin{itemize}
  \item Métodos de \textbf{Purged CV} y \textbf{Combinatorial Purged CV} se han desarrollado específicamente para ML financiero, para evitar \emph{look‑ahead bias} en estrategias temporales. [web:66][web:54]
  \item Ridge con selección de \( \alpha \) por CV temporal proporciona una referencia robusta de rendimiento, que se puede usar como “piso” para comparar modelos más complejos.
\end{itemize}
[web:67][web:69]

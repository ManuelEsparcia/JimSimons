\section{Módulo \texttt{models/inference/predict.py}}

\subsection{Objetivo}
El módulo \texttt{models/inference/predict.py} tiene como objetivo generar \textbf{predicciones de inferencia} (scores, probabilidades, retornos esperados) de forma:
\begin{itemize}
    \item \textbf{Reproducible}: trazabilidad completa desde \texttt{model\_ref} \(\leftrightarrow\) \texttt{model\_registry} \(\leftrightarrow\) \texttt{features\_store}.
    \item \textbf{Trazable}: metadatos de \texttt{run\_id}, \texttt{inference\_ts\_utc}, \texttt{model\_version}, \texttt{output\_schema\_version}.
    \item \textbf{Batch}: optimizado para procesar \texttt{date} \(\times\) \texttt{symbol} en grandes volúmenes.
    \item \textbf{Listo para consumo} por \texttt{portfolio/sizing}, \texttt{backtest/engine} y \texttt{risk/exposure\_report}, sin transformaciones manuales adicionales.
\end{itemize}

La salida típica de este módulo es crucial para estrategias de \textit{swing trading} de alta rotación, donde la calidad de la señal \texttt{y\_pred} y la robustez de la \texttt{feature\_pipeline} antes del punto de decisión determinan la \textit{alpha} realizable.

\subsection{Contexto de Batch Inference Industrial}
En sistemas de \textit{quant trading}, el \textbf{batch scoring} (batch inference) es el estándar de facto para generar señales en carteras amplias de activos \cite{mlbatchscoring2024azure}. El patrón habitual combina:
\begin{itemize}
    \item \textbf{Model registry} como fuente de verdad de \texttt{model\_version}.
    \item \textbf{Feature store} con \textit{point‑in‑time} \texttt{features} \cite{featurestore2025pointintime}.
    \item \textbf{Post‑processing configurable} (calibración, ranking, clipping) para adaptar la salida del modelo a la función de \textit{portfolio sizing} y \textit{risk management}.
\end{itemize}

La gestión estricta de \textit{data leakage} y \textit{temporal correctness} en el pipeline de inferencia es crítica para evitar rendimientos engañosos en backtesting y degradación de \textit{alpha} en producción \cite{leakage2022overview}.

\subsection{Alcance Funcional}

\subsubsection{Incluye}
\begin{itemize}
    \item \textbf{Carga de modelo versionado} desde \texttt{model\_registry} usando \texttt{model\_ref}.
    \item \textbf{Validación de compatibilidad} entre modelo y feature set:
    \begin{itemize}
        \item Compatibilidad de \texttt{feature\_set\_hash}, \texttt{schema} y \texttt{training\_feature\_list}.
    \end{itemize}
    \item \textbf{Predicción batch} por \texttt{date} y \texttt{symbol}:
    \[
    \big\{ (t, s) \in \mathcal{T} \times \mathcal{S} \big\} \mapsto \mathbf{\hat{y}}_{t,s} \in \mathbb{R}^{n}.
    \]
    \item \textbf{Post‑procesado de scores}:
    \begin{itemize}
        \item \texttt{clip} (e.g.\ percentiles \texttt{score\_clip\_percentiles}).
        \item \texttt{ranking} (e.g.\ normalización \texttt{y\_score\_rank}).
        \item \texttt{calibration} opcional (isotonic, Platt, o \texttt{calibrator\_ref} externo).
    \end{itemize}
    \item \textbf{Persistencia de outputs y metadata}:
    \begin{itemize}
        \item Outputs listos para \texttt{portfolio/sizing} y \texttt{backtest/engine}.
    \end{itemize}
\end{itemize}

\subsubsection{No incluye}
\begin{itemize}
    \item \textbf{Entrenamiento del modelo}: asume que el modelo ya está registrado en \texttt{model\_registry}.
    \item \textbf{Selección final de pesos de cartera}: decisión de \textit{position sizing} y \textit{optimización} se delega a módulos de cartera y risk.
    \item \textbf{Ejecución real de órdenes}: este módulo solo genera señales, no interactúa con brokers.
\end{itemize}

\subsection{Dependencias}

\subsubsection{Upstream}
\begin{itemize}
    \item \texttt{models/inference/model\_registry}: resolución de \texttt{model\_ref} \(\rightarrow\) \texttt{model\_version} + \texttt{artifact\_uri}.
    \item \texttt{features/store}: \texttt{features} versionados y \textit{point‑in‑time friendly}.
    \item \texttt{feature\_dictionary/lineage}: definición de \texttt{training\_feature\_list} y metadata de cada feature.
    \item \texttt{calibrator} opcional: modelo o función de calibración (e.g.\ \texttt{isotonic\_regression}, \texttt{Platt\_scaling}).
\end{itemize}

\subsubsection{Downstream}
\begin{itemize}
    \item \texttt{portfolio/sizing}: consume \texttt{y\_pred\_calibrated} o \texttt{y\_score\_rank} para generar \textit{weights}.
    \item \texttt{backtest/engine}: ejecuta backtesting y walk‑forward usando \texttt{scores.parquet}.
    \item \texttt{risk/exposure\_report}: analiza exposición y \textit{factor risk} basado en \texttt{y\_pred}.
\end{itemize}

\subsection{Contrato de Entrada}

\subsubsection{Entradas mínimas}
\begin{itemize}
    \item \texttt{model\_ref}: \texttt{model\_id} o \texttt{run\_id} + \texttt{fold/policy} para resolver la versión activa.
    \item \texttt{features\_path}: ruta a los features de inferencia (store parquet).
    \item \texttt{inference\_config\_path}: configuración de \texttt{predict.yaml}.
    \item \texttt{run\_id}: identificador único del experimento de inferencia.
\end{itemize}

\subsubsection{Entradas opcionales}
\begin{itemize}
    \item \texttt{calibration\_ref}: referencia a calibrador específico (e.g.\ \texttt{calibrator\_v1}).
    \item \texttt{date\_from} / \texttt{date\_to}: ventana de tiempo de inferencia.
    \item \texttt{universe\_filter\_path}: filtro de universo de activos (sector, tamaño, etc.).
\end{itemize}

\subsubsection{Campos mínimos esperados en features}
\begin{itemize}
    \item \texttt{date}, \texttt{symbol} como identificadores de tiempo y activo.
    \item \texttt{columnas de features requeridas} por el modelo, según \texttt{training\_feature\_list}.
\end{itemize}

\subsubsection{Parámetros clave}
\begin{itemize}
    \item \texttt{prediction\_mode}: \texttt{raw} | \texttt{calibrated} | \texttt{ranked}.
    \item \texttt{batch\_size}: tamaño de lote para procesamiento en memoria.
    \item \texttt{missing\_feature\_policy}: \texttt{fail} | \texttt{impute} | \texttt{drop}.
    \item \texttt{score\_clip\_percentiles}: percentiles para clipping robusto de scores extremos.
    \item \texttt{output\_schema\_version}: versión de esquema de salida para backward compatibility.
\end{itemize}

\subsection{Contrato de Salida}

\subsubsection{Persistencia}
\begin{itemize}
    \item \texttt{models/predictions/\{model\_id\}/\{run\_id\}/scores.parquet}:
    \begin{itemize}
        \item Salida principal de \texttt{y\_pred} por \texttt{date} y \texttt{symbol}.
    \end{itemize}
    \item \texttt{models/predictions/\{model\_id\}/\{run\_id\}/scores\_summary.json}:
    \begin{itemize}
        \item Estadísticas de distribución de \texttt{y\_pred} (media, std, percentiles, skewness, kurtosis).
    \end{itemize}
    \item \texttt{models/predictions/\{model\_id\}/\{run\_id\}/manifest.json}:
    \begin{itemize}
        \item Metadatos de \texttt{run\_id}, \texttt{model\_id}, \texttt{model\_version}, \texttt{inference\_ts\_utc}, \texttt{features\_path}, \texttt{calibration\_ref}, \texttt{batch\_size}.
    \end{itemize}
\end{itemize}

\subsubsection{Campos mínimos en \texttt{scores.parquet}}
\begin{itemize}
    \item \texttt{date}, \texttt{symbol}.
    \item \texttt{model\_id}, \texttt{model\_version}.
    \item \texttt{y\_pred\_raw}: predicción cruda del modelo.
    \item \texttt{y\_pred\_calibrated} (opcional): calibración probabilística.
    \item \texttt{y\_score\_rank} (opcional): ranking de scores dentro de la ventana de tiempo.
    \item \texttt{inference\_ts\_utc}: timestamp de ejecución de inferencia.
    \item \texttt{run\_id}.
\end{itemize}

\subsection{Reglas de PIT y Anti‑Leakage}

\begin{itemize}
    \item \textbf{Únicamente features disponibles a \texttt{decision\_cut}}:
    \begin{itemize}
        \item Cada \texttt{feature} debe computarse exclusivamente a partir de información \(\leq t\) en inferencia.
        \item Evita \textit{look‑ahead} y \textit{target leakage} en señales de retorno \cite{leakage2022overview}.
    \end{itemize}
    \item \textbf{Prohibido usar columnas no presentes en \texttt{training\_feature\_list}}:
    \begin{itemize}
        \item Implementa \textit{feature equality} rigurosa para evitar \textit{feature leakage} \cite{leakage2024time}.
    \end{itemize}
    \item \textbf{Calibración fitting en datos pasados al periodo de inferencia}:
    \begin{itemize}
        \item Si se usa calibrador, debe ser ajustado en \texttt{t\_train} \(\ll\) \texttt{t\_inference}.
    \end{itemize}
    \item \textbf{No mezclar modelos de fecha futura para inferencia histórica}:
    \begin{itemize}
        \item Evitar \textit{temporal misalignment} entre \texttt{model\_version} y \texttt{features\_path}.
    \end{itemize}
\end{itemize}

Estas reglas siguen el patrón de \textit{point‑in‑time} correctness descrito en \cite{featurestore2025pointintime} y recomendaciones de \textit{data leakage} en sistemas de time series \cite{leakage2022overview}.

\subsection{Flujo de Predicción}

\begin{enumerate}
    \item \textbf{Cargar \texttt{inference\_config} y resolver \texttt{model\_ref}} en \texttt{model\_registry}:
    \begin{itemize}
        \item \texttt{resolve\_model\_ref\_from\_registry(...)}.
    \end{itemize}
    \item \textbf{Cargar artefacto de modelo + metadata}:
    \begin{itemize}
        \item Comprobar \texttt{model\_version}, \texttt{feature\_set\_hash}, \texttt{code\_commit}.
    \end{itemize}
    \item \textbf{Cargar features} y filtrar \texttt{date\_from} \(\rightarrow\) \texttt{date\_to} y \texttt{universe\_filter}:
    \begin{itemize}
        \item Lectura \textit{lazy} desde \texttt{features/store} optimizada para filtros por \texttt{date} y \texttt{symbol}.
    \end{itemize}
    \item \textbf{Validar schema y feature order}:
    \begin{itemize}
        \item Verificar coincidencia con \texttt{training\_feature\_list} y \texttt{expected\_dtype}.
    \end{itemize}
    \item \textbf{Aplicar política de features faltantes}:
    \begin{itemize}
        \item \texttt{missing\_feature\_policy=fail} bloquea ejecución si hay columnas ausentes.
        \item \texttt{impute} reemplaza con valores predefinidos (media/mode).
        \item \texttt{drop} elimina instancias con features faltantes.
    \end{itemize}
    \item \textbf{Ejecutar prediccion por lotes}:
    \begin{itemize}
        \item Procesamiento \texttt{batch} con \texttt{batch\_size} configurable.
    \end{itemize}
    \item \textbf{Aplicar post‑procesado}:
    \begin{itemize}
        \item \texttt{clip} basado en \texttt{score\_clip\_percentiles}.
        \item \texttt{ranking} (e.g.\ \texttt{y\_score\_rank = rank(y\_pred\_raw)}).
        \item \texttt{calibration} opcional (isotonic/Platt o \texttt{calibrator\_ref}).
    \end{itemize}
    \item \textbf{Persistir scores + resumen + manifest}:
    \begin{itemize}
        \item \texttt{persist\_prediction\_outputs(...)}.
    \end{itemize}
\end{enumerate}

Este flujo se alinea con estándares de \textit{batch inference pipelines} en MLOps \cite{mlbatchscoring2024azure}.

\subsection{Validaciones Mínimas}

\begin{itemize}
    \item \textbf{Features requeridas presentes o gestionadas por \texttt{missing\_feature\_policy}}.
    \item \textbf{Tipos numéricos compatibles} con \texttt{model\_input\_dtype}.
    \item \textbf{Sin \texttt{NaN}/\texttt{Inf} en salida final}:
    \begin{itemize}
        \item \texttt{isnan(\mathbf{\hat{y}}) \equiv \emptyset}.
    \end{itemize}
    \item \textbf{Cobertura de predicción por fecha \(\geq\) umbral}:
    \begin{itemize}
        \item \texttt{pct\_covered\_by\_date} debe ser \(\geq\) \texttt{min\_coverage\_pct}.
    \end{itemize}
    \item \textbf{Consistencia de \texttt{model\_id}/\texttt{version} en todos los registros}:
    \begin{itemize}
        \item Evitar \textit{model drift} espurio durante la inferencia.
    \end{itemize}
\end{itemize}

\subsection{Manejo de Errores}

\begin{itemize}
    \item \textbf{\texttt{model\_ref} inexistente} \(\rightarrow\) \texttt{fail} inmediato.
    \item \textbf{Schema mismatch severo} \(\rightarrow\) \texttt{fail} (o \texttt{warn} si la política lo permite):
    \begin{itemize}
        \item \texttt{missing\_feature\_policy=fail} \(\rightarrow\) \texttt{validate\_feature\_schema} falla.
    \end{itemize}
    \item \textbf{Porcentaje alto de imputación} \(\rightarrow\) \texttt{warn/fail} por umbral:
    \begin{itemize}
        \item \texttt{pct\_imputed\_features} \(\geq\) \texttt{max\_pct\_imputed} \rightarrow \texttt{WARNING}.
    \end{itemize}
    \item \textbf{Lote fallido} \(\rightarrow\) \texttt

Modulo: models/inference/predict.py

1) Objetivo
Generar predicciones de inferencia (scores/probabilidades/retornos esperados) de forma reproducible y trazable, usando modelos registrados y features PIT, listas para consumo por portfolio/backtest.

2) Alcance
Incluye:
- Carga de modelo versionado desde registry.
- Validacion de compatibilidad entre modelo y feature set.
- Prediccion batch por fecha/simbolo.
- Post-procesado de scores (clipping, ranking, calibration opcional).
- Persistencia de outputs y metadata de inferencia.

No incluye:
- Entrenamiento del modelo.
- Seleccion final de pesos de cartera.
- Ejecucion real de ordenes.

3) Dependencias
Upstream:
- models/inference/model_registry
- features/store
- feature_dictionary/lineage
- calibrator opcional

Downstream:
- portfolio/sizing
- backtest/engine
- risk/exposure_report

4) Contrato de entrada
Entradas minimas:
- model_ref (model_id o run_id + fold/policy)
- features_path
- inference_config_path
- run_id

Opcionales:
- calibration_ref
- date_from/date_to
- universe_filter_path

Campos minimos esperados:
- date, symbol
- columnas de features requeridas por el modelo

Parametros clave:
- prediction_mode: raw | calibrated | ranked
- batch_size
- missing_feature_policy: fail | impute | drop
- score_clip_percentiles
- output_schema_version

5) Contrato de salida
- models/predictions/{model_id}/{run_id}/scores.parquet
- models/predictions/{model_id}/{run_id}/scores_summary.json
- models/predictions/{model_id}/{run_id}/manifest.json

Campos minimos:
- date, symbol
- model_id, model_version
- y_pred_raw
- y_pred_calibrated (opcional)
- y_score_rank (opcional)
- inference_ts_utc
- run_id

6) Reglas PIT y anti-leakage
- Solo features disponibles a decision_cut.
- Prohibido usar columnas no presentes en training_feature_list.
- Si hay calibration, debe ser fitted en datos pasados al periodo de inferencia.
- No mezclar modelos de fecha futura para inferencia historica.

7) Flujo del proceso
1. Cargar config y resolver model_ref en registry.
2. Cargar artefacto de modelo + metadata.
3. Cargar features y filtrar rango/universo.
4. Validar schema y feature order.
5. Aplicar politica de missing features.
6. Ejecutar prediccion por lotes.
7. Aplicar post-procesado (clip/rank/calibration).
8. Persistir scores + resumen + manifest.

8) Validaciones minimas
- todas las features requeridas presentes o gestionadas por policy.
- tipos numericos compatibles con modelo.
- sin NaN/inf en salida final.
- cobertura de prediccion por fecha >= umbral.
- consistencia de model_id/version en todos los registros.

9) Manejo de errores
- model_ref inexistente -> fail inmediato.
- schema mismatch severo -> fail (o warn si policy lo permite).
- porcentaje alto de imputacion -> warn/fail por umbral.
- lote fallido -> retry limitado y logging de detalle.

10) Observabilidad
Logging:
- INFO: progreso por batch/fecha.
- WARNING: imputaciones altas o cobertura baja.
- ERROR: error de carga de modelo/schema.

Metricas:
- n_rows_scored
- n_symbols_scored
- pct_missing_imputed
- latency_total_sec
- latency_per_1k_rows

11) CLI sugerida
python -m models.inference.predict ^
  --model-ref lgbm_main_v3 ^
  --features-path features/store ^
  --config-path configs/inference/predict.yaml ^
  --date-from 2025-01-01 ^
  --date-to 2026-03-01 ^
  --run-id predict_20260305_001

12) Estructura interna recomendada
- load_inference_config(...)
- resolve_model_ref_from_registry(...)
- load_model_artifact(...)
- load_features_for_inference(...)
- validate_feature_schema(...)
- apply_missing_feature_policy(...)
- predict_batches(...)
- postprocess_scores(...)
- persist_prediction_outputs(...)
- main()

13) Criterios de aceptacion
- Predicciones reproducibles y trazables.
- Compatibilidad modelo-features validada.
- Output listo para portfolio/backtest sin transformaciones manuales.

14) Casos de prueba minimos
- prediccion baseline con modelo valido.
- fallo controlado por feature faltante (policy=fail).
- imputacion funciona cuando policy=impute.
- ranking/calibration opcionales producen columnas esperadas.

15) Checklist rapido
- [ ] Resolver modelo desde registry.
- [ ] Validar schema de features.
- [ ] Prediccion batch robusta.
- [ ] Post-procesado configurable.
- [ ] Outputs + manifest completos.

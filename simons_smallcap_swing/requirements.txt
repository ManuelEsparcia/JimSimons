Archivo: requirements.txt (gestion de dependencias Python)

1) Objetivo
Definir dependencias runtime y de desarrollo de forma reproducible, minimizando conflictos de version y garantizando compatibilidad con el pipeline cuantitativo.

2) Politica de versionado
- pin estricto para paquetes criticos numericos.
- rangos acotados para paquetes de tooling no criticos.
- lockfile recomendado (poetry.lock ya presente).

3) Grupos recomendados
Runtime core:
- numpy, pandas, scipy, pyarrow
- scikit-learn, xgboost, lightgbm
- statsmodels, pydantic

Research/validation:
- mlflow
- shap
- matplotlib/seaborn

Testing/quality:
- pytest
- pytest-cov
- mypy/ruff (si se usan)

4) Reglas
- cualquier cambio en dependencias requiere:
  - actualizar lockfile.
  - registrar motivo en changelog interno.
  - ejecutar suite minima de tests.

5) Compatibilidad sugerida
- version minima Python coherente con pyproject.
- validar wheels disponibles para entorno objetivo.

6) DoD
- dependencias declaradas, reproducibles y auditables para CI/local.

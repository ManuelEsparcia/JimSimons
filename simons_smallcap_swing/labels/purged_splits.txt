\section{Módulo: \texttt{labels/purged\_splits.py}}

\subsection{Objetivo del módulo}

El objetivo de \texttt{labels/purged\_splits.py} es generar \emph{particiones temporales} de entrenamiento, validación y test para un panel \((date, symbol)\) de small caps, garantizando:

\begin{itemize}
  \item \textbf{Sin leakage} mediante \emph{Purged K‑Fold} + \emph{Embargo} \cite{LopezdePrado2018,PurgedCV2017}.
  \item Respeto a los \emph{horizontes forward} de los labels y a las ventanas de información de las features.
  \item Soporte tanto para \emph{cross‑validation temporal} (\texttt{purged\_kfold}) como para \emph{walk‑forward} designs.
  \item Exportación de \emph{índices/splits} reproducibles para \texttt{models/*/train\_*.py} y \texttt{backtest/run\_walkforward.py}.
\end{itemize}

Este módulo implementa el esquema estándar de \emph{temporal cross‑validation} industrial para \emph{swing trading strategies} basadas en ML \cite{LopezdePrado2018,BacktestOverfitting2024}.

\subsection{Marco matemático de purged splits}

\subsubsection{Definiciones de tiempo y evento}

Denotamos:

\begin{itemize}
  \item \( t \): \emph{trade date} o \emph{decision date} de la observación \((t, i)\).
  \item \( h \): horizonte forward del label (por ejemplo, \( h = 5d, 10d, 20d \)).
  \item \( \mathcal{H} \): conjunto de horizontes disponibles.
\end{itemize}

\textbf{Intervalo de evento} asociado a \((t, i)\):

\begin{equation}
\mathcal{I}_{t,i} = [t_{\text{start}}, t_{\text{end}}], \quad
\text{donde } t_{\text{start}} = t + \text{lag},\ t_{\text{end}} = t + \text{lag} + h.
\end{equation}

En presencia de \emph{multi‑horizonte}, se usa el horizonte primario o \(\max(\mathcal{H})\) como \texttt{primary\_horizon\_days} para definir \(\mathcal{I}_{t,i}\) de forma conservadora.

\subsubsection{Purga (Purge)}

Sea \(\mathcal{D}_{\text{test}}\) el conjunto de fechas de test en un fold. Se define el \emph{event window} de test:

\begin{equation}
\mathcal{W}_{\text{test}} = \bigcup_{(t,i) \in \mathcal{D}_{\text{test}}} \mathcal{I}_{t,i}.
\end{equation}

Una observación \((t,i)\) en \emph{train} se marca como \texttt{purged} si:

\begin{equation}
\mathcal{I}_{t,i} \cap \mathcal{W}_{\text{test}} \neq \emptyset.
\end{equation}

Esto asegura que ningún \emph{label window} de train se solape con el \emph{event interval} de test \cite{LopezdePrado2018,PurgedCV2017}.

\subsubsection{Embargo temporal}

Además de la purga, se aplica un \emph{embargo} en \emph{calendario de trading} tras el bloque de test:

\begin{equation}
\mathcal{E}_{\text{test}} = \{ t \in \mathcal{T} : t > \max(\mathcal{D}_{\text{test}}) \text{ y } t \leq \max(\mathcal{D}_{\text{test}}) + \text{embargo\_days} \},
\end{equation}

y:

\begin{equation}
\text{is\_embargoed}(t,i) = \mathbb{I}\{ t \in \mathcal{E}_{\text{test}} \}.
\end{equation}

El embargo previene \emph{serial dependence} y \emph{indirect leakage} a través de series temporales de features de ventana (medias móviles, volatilidades, z‑scores, etc.) \cite{LopezdePrado2018,PurgedCV2017}.

\subsubsection{Construcción de folds temporales}

Sea \(\mathcal{T} = \{t_1, t_2, \dots, t_N\}\) el conjunto de fechas únicas en \([start\_date, end\_date]\). El módulo implementa:

\begin{itemize}
  \item \textbf{Purged K‑Fold}:
  \begin{enumerate}
    \item Dividir \(\mathcal{T}\) en \(K\) bloques contiguos (test folds).
    \item Para cada fold \(k\):
    \begin{itemize}
      \item Definir \(\mathcal{D}_{\text{test}}^{(k)}\).
      \item Calcular \(\mathcal{W}_{\text{test}}^{(k)}\).
      \item Calcular \(\mathcal{E}_{\text{test}}^{(k)}\).
      \item Eliminar de train todas las observaciones \((t,i)\) tales que \(\mathcal{I}_{t,i} \cap \mathcal{W}_{\text{test}}^{(k)} \neq \emptyset\) o \(t \in \mathcal{E}_{\text{test}}^{(k)}\).
    \end{itemize}
  \end{enumerate}
  \item \textbf{Walk‑forward}:
  \begin{enumerate}
    \item Mover una ventana \emph{creciente} o \emph{rodante} de train.
    \item El siguiente bloque corresponde a \emph{valid} o \emph{test}.
    \item Aplicar purge + embargo entre cada par \((train, test)\).
  \end{enumerate}
\end{itemize}

Esta estructura está alineada con \emph{Combinatorial Purged Cross‑Validation (CPCV)} para \emph{panel data} financiero \cite{LopezdePrado2018,CombinatorialPurged2018}.

\subsection{Flujo del proceso (alto nivel)}

\begin{enumerate}
  \item \texttt{load\_labels} desde \texttt{labels\_path} con \texttt{label\_valid\_flag} y \texttt{horizon\_used}.
  \item \texttt{load\_split\_config} y \texttt{start\_date}/\texttt{end\_date}.
  \item \texttt{build\_event\_intervals(df\_labels, primary\_horizon\_days, lag\_days)}.
  \item \texttt{make\_time\_folds(dates, cfg)}:
  \begin{itemize}
    \item \texttt{split\_mode = purged\_kfold} o \texttt{walkforward}.
  \end{itemize}
  \item Para cada fold:
  \begin{itemize}
    \item \texttt{apply\_purge(train\_df, test\_df, event\_intervals, cfg)}.
    \item \texttt{apply\_embargo(train\_df, test\_dates, calendar, cfg)}.
    \item \texttt{validate\_no\_leakage(train\_df, test\_df, event\_intervals)}.
  \end{itemize}
  \item \texttt{build\_fold\_outputs} y \texttt{persist\_splits\_outputs}.
\end{enumerate}

Este flujo garantiza \emph{reproducibilidad} y \emph{trazabilidad} de \texttt{run\_id} y \texttt{config\_hash} \cite{LopezdePrado2018}.

\subsection{Entradas y salidas}

\subsubsection{Entradas}

\begin{itemize}
  \item \texttt{labels\_path}: \texttt{labels/store} (con \texttt{date}, \texttt{symbol}, \texttt{label\_valid\_flag}, \texttt{horizon\_used} o \texttt{horizons}).
  \item \texttt{start\_date}, \texttt{end\_date}: rango temporal de splits.
  \item \texttt{split\_config\_path}: \texttt{embargo\_pct} o \texttt{embargo\_days}, \texttt{n\_splits}, \texttt{min\_train\_days}, etc.
  \item \texttt{run\_id}.
  \item Opcionalmente \texttt{features\_path} para validar intersección exacta train/test.
\end{itemize}

\subsubsection{Salidas}

\begin{itemize}
  \item \texttt{labels/splits/purged\_splits_{run\_id}.parquet}:
  \begin{itemize}
    \item \texttt{fold\_id}, \texttt{split\_role} (\texttt{train}/\texttt{valid}/\texttt{test}).
    \item \texttt{date}, \texttt{symbol}, \texttt{sample\_index}.
    \item \texttt{is\_purged}, \texttt{is\_embargoed}, \texttt{inclusion\_flag}.
  \end{itemize}
  \item \texttt{labels/splits/purged\_splits\_summary_{run\_id}.json}:
  \begin{itemize}
    \item \texttt{n\_splits}.
    \item \texttt{train\_start}, \texttt{train\_end}, \texttt{test\_start}, \texttt{test\_end} por fold.
    \item \texttt{leakage\_check\_pass} booleano por fold.
  \end{itemize}
  \item \texttt{labels/splits/fold\_k\_indices_{run\_id}.parquet} para cada \(k\).
\end{itemize}

\subsection{Reglas PIT y anti‑leakage}

\begin{itemize}
  \item \textbf{Purge} se calcula usando el \emph{event end} real de cada observación \((t + h)\), no fechas arbitrarias:
  \begin{equation}
  t_{\text{end}} = t + \text{lag} + h.
  \end{equation}
  \item En \emph{multi‑horizonte}, se puede usar \texttt{primary\_horizon\_days} o \(\max(\mathcal{H})\) como \emph{conservative horizon}.
  \item \textbf{Embargo} se aplica en \emph{calendario de trading} (no días naturales), respetando \texttt{gaps\_dias} configurables.
  \item Ninguna fecha de train puede caer en \texttt{purged} o \texttt{embargo} de test:
  \begin{equation}
  \nexists t \in \mathcal{T}_{\text{train}} \text{ tal que } t \in \mathcal{W}_{\text{test}} \cup \mathcal{E}_{\text{test}}.
  \end{equation}
\end{itemize}

\subsection{Validaciones y métricas de calidad}

\subsubsection{Validaciones mínimas}

\begin{itemize}
  \item \textbf{Integridad}:
  \begin{itemize}
    \item Sin duplicados en \((fold\_id, split\_role, date, symbol)\).
    \item \texttt{inclusion\_flag} coherente con \texttt{is\_purged} y \texttt{is\_embargoed}.
  \end{itemize}
  \item \textbf{No‑leakage}:
  \begin{itemize}
    \item \texttt{leakage\_check\_pass} = true para todos los folds.
    \item \(\mathcal{I}_{t,i} \cap \mathcal{W}_{\text{test}} = \emptyset\) para todo \texttt{train} incluido.
  \end{itemize}
  \item \textbf{Cobertura}:
  \begin{itemize}
    \item \texttt{n\_train\_final} y \texttt{n\_test} por fold \(\geq\) mínimos configurados.
  \end{itemize}
\end{itemize}

\subsubsection{Métricas de observabilidad}

\begin{itemize}
  \item \texttt{pct\_removed\_by\_purge}, \texttt{pct\_removed\_by\_embargo}.
  \item \texttt{avg\_train\_size}, \texttt{avg\_test\_size}.
  \item \texttt{leakage\_violations\_count}.
  \item \texttt{run\_duration\_sec}.
\end{itemize}

Ejemplo de CLI:

\begin{verbatim}
python -m labels.purged_splits \
  --labels-path labels/store \
  --start-date 2014-01-01 \
  --end-date 2026-03-01 \

\subsection{Criterios de aceptación (Definition of Done)}

\begin{itemize}
  \item \textbf{Splits reproducibles y sin leakage}:
  \begin{itemize}
    \item \texttt{leakage\_check\_pass} = \texttt{True} en todos los folds.
    \item Purge y embargo aplicados correctamente según \texttt{embargo\_pct} o \texttt{embargo\_days}.
  \end{itemize}
  \item \textbf{Balanced design}:
  \begin{itemize}
    \item \texttt{min\_train\_days}, \texttt{min\_test\_days} cumplidos.
  \end{itemize}
  \item \textbf{Integrabilidad}:
  \begin{itemize}
    \item \texttt{fold\_k\_indices} compatibles con \texttt{models/*/train\_*.py} y \texttt{backtest/run\_walkforward.py}.
  \end{itemize}
\end{itemize}

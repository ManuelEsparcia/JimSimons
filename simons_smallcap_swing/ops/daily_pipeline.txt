\section{Introducción y Contexto Estratégico}
Este módulo orquesta el pipeline diario completo de un sistema de \textbf{swing trading cuantitativo}, desde ingesta de datos hasta backtest y controles de riesgo. En swing trading, capturamos movimientos de 2-10 días con Sharpe ratios >2, inspirado en Renaissance Technologies (Jim Simons), donde pipelines robustos generan alpha consistente vía reproducibilidad y fault-tolerance [web:44][web:49].

El objetivo es maximizar rentabilidad: $\alpha = E[R_p - R_b] - \lambda \sigma_p$, con $\lambda$ penalizador de riesgo, optimizado bajo constraints de liquidez y drawdown <15\% [web:47]. No incluye lógica de features/ML (modular), pero integra con ellas para daily rebalance.

\section{Matemáticas Fundamentales del Pipeline}
El DAG lógico modela dependencias como grafo acíclico dirigido (DAG): $G=(V,E)$, $V$=etapas (data$\to$portfolio), $E$=precedencias. Topological sort vía Kahn's algorithm asegura orden: $\text{indegree}(v)=0 \implies \text{queue}$.

Tiempos/SLA: modelados como $T_{total} = \sum T_i + \sum W_{ij}$, con $W_{ij}$ esperas por dependencias. Retry policy: backoff exponencial $B_k = B_0 \cdot \gamma^k$, $\gamma=2$, $k \leq \max_retries$ para fallos transitorios (e.g., API rates) [web:51].

Gates críticos: umbrales probabilísticos, e.g., data quality $Q = 1 - \frac{\sum |m_i - \hat{m}_i|}{\sum |m_i|}$, $Q > 0.95$ o fail-fast.

Idempotencia vía hashing: $\text{config\_hash} = \text{SHA256}(\text{yaml} + \text{execution\_date})$, evita re-runs corruptos.

\begin{lstlisting}[language=Python]
def main(config_path, exec_date, run_id):
    cfg = load_config(config_path)
    plan = build_dag(plan_yaml)  # NetworkX DiGraph
    context = PipelineContext(run_id, exec_date)
    
    preflight_ok = run_preflight(context)  # Disk, creds, market open
    if not preflight_ok: raise PipelineAbort
    
    for step in topological_sort(plan):
        for attempt in range(cfg.max_retries):
            try:
                output = execute_step(step, context)
                persist_state(step, 'success', output)
                break
            except TransientError:
                backoff_sleep(2**attempt)
        else:
            persist_state(step, 'fail')
            if cfg.fail_fast: propagate_fail(step)
    
    postflight = run_postflight(context)  # Coherencia global
    emit_summary(run_id, postflight.metrics)
\end{lstlisting}

\section{Backtest Ultra-Optimizado (Extensión SOTA)}
Integra Backtesting.py [web:51] en final step para validación quick:
\begin{lstlisting}[language=Python]
from backtesting import Backtest, Strategy
class SwingQuantStrat(Strategy):
    def init(self):
        self.ma = self.I(SMA, self.data.Close, 10)
    def next(self):
        if self.data.Close[-1] > self.ma[-1]: self.buy()

bt = Backtest(data, SwingQuantStrat, cash=1e6)
stats = bt.run()  # Sharpe, Win Rate, etc.
assert stats['Sharpe Ratio'] > 2.0  # Gate
\end{lstlisting}

Añade slippage: $P_{exec} = P_{mid} \pm 0.5 \cdot \sigma \sqrt{V}$, costs modelados. Walk-forward para evitar lookahead bias [web:49].

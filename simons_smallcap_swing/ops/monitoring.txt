\section{Introducción y Contexto Estratégico}
Monitoreo unifica signals de pipeline/data/drift/risk para MTTR<1h, esencial en swing donde delays cuestan $\Delta P = \sigma \sqrt{\Delta t}$ (vol drag). Taxonomía accionable mantiene uptime>99.5\%, alpha preservado [web:47][web:49].

Integra upstream (daily\_pipeline, data\_health) $\to$ downstream (Slack/email), como Prometheus+Grafana en hedge funds.

\section{Matemáticas Fundamentales del Monitoreo}
\textbf{SLA Breach}: $B = \max(0, T_{actual} - T_{SLA})$, alert si $B > \tau$.

\textbf{Alert Severity Score}: $S = w_p P_f + w_d D + w_r R_{dd} + w_m M_{drift}$, $\sum w=1$, e.g., $w_r=0.4$ (risk domina).

\textbf{Drawdown Alert}: $DD_t = \max(0, 1 - \frac{P_t}{P^*})$, $P^*=\max P$, critical si $DD>10\%$.

\textbf{MTTR Proxy}: $MTTR = \frac{\sum t_{close} - t_{open}}{N_{critical}}$, optimizar <30min.

Dedupe: Jaccard similarity $J(A,B) = \frac{|A\cap B|}{|A\cup B|} > 0.8 \implies$ merge.

\textbf{Flujo Pseudocódigo Optimizado}:
\begin{lstlisting}[language=Python]
def main(config_path, run_id, exec_date):
    cfg = load_config(config_path)
    sources = {
        'pipeline': load_json(f'ops/runs/{run_id}/summary.json'),
        'data_health': load_parquet(f'ops/data_health/{run_id}/summary.parquet'),
        # + drift, risk, validation
    }
    
    events = normalize_events(sources)
    alerts = []
    for event in events:
        severity = score_severity(event, cfg.rules)  # 'critical' if DD>10%
        alert = {'id': uuid(), 'type': event['type'], 'severity': severity,
                 'action': recommend_action(severity, event)}
        if not dedupe(alert, recent_alerts):  # Jaccard >0.8
            alerts.append(alert)
            dispatch_notification(alert, cfg.channels)
    
    snapshot = {'alerts': alerts, 'sla_breaches': compute_sla(sources)}
    persist_parquet(snapshot, run_id)
    return snapshot
\end{lstlisting}

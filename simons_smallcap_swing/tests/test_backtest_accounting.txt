\section{Objetivo Estratégico}
Validar consistencia contable del motor de backtest: cash, posiciones, NAV, PnL y costes para estrategias swing (holding 3-10 d\'ias). Asegura reconciliaci\'on exacta $\NAV_t = \cash_t + \sum_i \pos_i \cdot \price_{t,i}$ bajo tolerancia $10^{-6}$, previniendo leaks que inflan Sharpe $\Sharpe = \frac{\E[\PnL] - r_f}{\sigma(\PnL)}$. Cr\'itico para auditor\'ia industrial como en Medallion Fund.[web:56][web:54]

\section{Casos M\'inimos y Assertions Matem\'aticas}
\begin{itemize}
\item \texttt{test\_nav\_reconciliation\_cash\_plus\_positions}: $\NAV_t \approx \cash_t + \mtm_t$, MTM via $V_t = S_t \cdot N_t$ (Itô lemma para opciones si aplica).
\item \texttt{test\_gross\_to\_net\_pnl\_waterfall\_consistency}: $\PnLnet = \PnLgross - \sum (\commission + \slippage + \impact)$, waterfall descompuesto.
\item \texttt{test\_partial\_fills\_update\_positions\_correctly}: $\pos_{t+1} = \pos_t + \fillqty \cdot \sign(order)$, sin overshoot.
\item \texttt{test\_cost\_components\_non\_negative}: $\forall c \in \{\commission, \slippage\} \geq 0$.
\item \texttt{test\_corporate\_action\_adjustment\_consistency}: Splits/dividends: $S_{adj} = S \cdot f$, $\cash += N \cdot div$.
\end{itemize}

\section{Matemáticas Avanzadas: Accounting bajo Procesos Estocásticos}
NAV evolution: $d\NAV_t = d\cash_t + d\mtm_t$, con $d\mtm_t = \sum (\delta_t dS_t + \frac{1}{2} \Gamma_t (dS_t)^2)$ (Greeks para non-linear). Costes: $\commission = \gamma \cdot \notional$, $\slippage = \lambda \sqrt{v_t}$, modelo Kyle. Gross PnL: $\PnLgross_t = \sum (exit - entry) \cdot shares$. SOTA: Incorporar transaction cost models de Almgren-Chriss $\cost = \eta \sigma \sqrt{\frac{X}{V}}$.[web:49][web:51]

Para swing: Waterfall optimizado minimizando TC impact en holds cortos, target $\PnLnet / \PnLgross > 0.85$.

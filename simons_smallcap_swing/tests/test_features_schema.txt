\section{Objetivo Estratégico}
Validar schema y calidad estructural del feature store, previniendo roturas en entrenamiento/inferencia para modelos swing (e.g., LSTM/Transformers prediciendo $r_{t:t+5}$). Detecta drift temprano, asegurando PK únicos y dtypes estables. Esencial para pipelines MLOps en fondos quant, donde schema mismatch cuesta millones en downtime.[web:54][web:56]

\section{Casos Mínimos y Assertions Estructurales}
\begin{itemize}
\item \texttt{test\_features\_required\_columns\_present}: Core: date, symbol, run\_id, config\_hash.
\item \texttt{test\_features\_primary\_key\_unique\_date\_symbol}: No duplicados $(date, symbol)$.
\item \texttt{test\_features\_dtypes\_match\_schema}: Compatibles con \texttt{simons\_core.schemas}.
\item \texttt{test\_no\_inf\_values\_in\_numeric\_features}: $\max(|f_i|) < \infty$.
\item \texttt{test\_feature\_coverage\_above\_threshold}: Coverage $>95\%$.
\end{itemize}

\section{Matemáticas Avanzadas: Feature Engineering Seguro}
Normalizaci\'on z-score: $f'_i = \frac{f_i - \mu_i}{\sigma_i}$, $\mu_i, \sigma_i$ rolling PIT. PCA: $\max \Var(X') s.t. \|X'\| \leq k$, eigenvalues de covarianza. Schema asegura no NaN/Inf post-transform: $f \in \mathbb{R}^p$, $p=50-200$ features (RSI, BBands, volume profiles). Drift detection: KS-test $D = \sup |F_1 - F_2| < 0.05$.[web:54]

Para swing: Features multitimeframe $f_{t}^{(h)} = \agg(P_{t-h:t})$, $h=\{1,5,20\}$ días.

\section{Objetivo y Contexto Industrial}
Este módulo centraliza utilidades de optimización numérica convexa y QP (Quadratic Programming) para construcción de portafolios en swing trading. En quant finance, la optimización resuelve:
\[
\min_w \frac{1}{2} w^T Q w + c^T w \quad \text{s.t.} \quad A w \geq b, \, G w = h, \, lb \leq w \leq ub
\]
donde $Q = \Sigma$ (covarianza), $c$ penaliza biases, constraints incluyen retorno objetivo, turnover, y long-only (simplex). Usado en Renaissance/Medallion para mean-variance con risk parity [Maillard et al., 2010].

\textbf{Referencias clave:}
\begin{itemize}
\item Markowitz (1952): ``Portfolio Selection'' -- QP origen.
\item Black-Litterman (1992): Views + priors.
\item Libro: \textit{Quantitative Equity Portfolio Management} (Chincarini \& Kim, 2021).
\end{itemize}

Estado del arte: Integración CVXPY + ECOS solver; extensiones a quantum QP via Qiskit [Perdomo-Ortiz et al., 2012].

\section{Alcance y Funciones Clave}

\subsection{solve\_qp(Q, c, A, b, G, h, bounds, solver\_cfg)}
Wrapper unificado para QP. Formulación estándar:
\begin{align}
Q &\succeq 0 \quad (\text{convexidad PSD}), \\
A &\in \mathbb{R}^{m \times n} \quad (\text{lineales $\geq$}), \\
G &\in \mathbb{R}^{p \times n} \quad (\text{igualdades}).
\end{align}
Manejo: OSQP fallback si CVXOPT falla. Tolerancias $\epsilon = 10^{-6}$.

\textbf{Código sugerido (Python):}
\begin{lstlisting}[language=Python, basicstyle=\ttfamily\small]
import cvxpy as cp
import numpy as np
from typing import Dict, Any

def solve_qp(Q: np.ndarray, c: np.ndarray, A: np.ndarray, b: np.ndarray,
             G: np.ndarray, h: np.ndarray, bounds: tuple = None,
             solver_cfg: Dict[str, Any] = None) -> Dict[str, Any]:
    n = Q.shape[0]
    w = cp.Variable(n)
    objective = cp.quad_form(w, Q)/2 + c @ w
    constraints = [A @ w >= b, G @ w == h]
    if bounds:
        lb, ub = bounds
        constraints += [w >= lb, w <= ub]
    prob = cp.Problem(cp.Minimize(objective), constraints)
    prob.solve(**solver_cfg or {'solver': 'ECOS'})
    return {
        'x_opt': w.value,
        'status': prob.status,
        'objective_value': prob.value,
        'diagnostics': {'residuals': prob.constraints[0].residuals()}
    }
\end{lstlisting}

\subsection{project\_to\_simplex(w)}
Proyección $\ell_1=1$, $w \geq 0$ (long-only portfolios):
\[
\pi(w) = \arg\min_{\hat{w} \geq 0, \sum \hat{w}=1} \| \hat{w} - w \|^2_2
\]
Solución closed-form: sort + cumulative [Michelot, 1986].

Estado del arte: Usado en risk budgeting [Roncalli, 2013].

\subsection{enforce\_box\_constraints(w, lb, ub)}
Clipping simple: $w_i = \max(lb_i, \min(ub_i, w_i))$. Post-procesado para regulatory (e.g., 10\% max weight).

\subsection{build\_turnover\_penalty(w_old, lambda=0.1)}
Penalidad L1: $c = \lambda |w - w_{old}|_1$ para low-turnover en swing (reduce costs ~0.1\%/trade).

\section{Diagnósticos Avanzados: Condiciones KKT}
Verificación primal-dual optimality:
\begin{align}
\nabla L(w^*, \lambda^*, \nu^*) &= 0, \\
A w^* &\geq b \quad \Rightarrow \quad \lambda^* \geq 0, \, \lambda^{*T} (A w^* - b) = 0.
\end{align}
Función \texttt{check\_kkt\_conditions}: Computa residuals $\|r_p\| < \epsilon$, $\|r_d\| < \epsilon$.

Feasibility report: Detección infactible via dual unbounded.

\section{Integración en Swing Trading}
- \texttt{portfolio/optimizer}: QP para weights semanales (swing horizon).
- Calibración blender: Opt multi-estrategia.
- Backtest: Incorpora costs reales, slippage; target Sharpe $>2.0$.

\textbf{Ejemplo toy (Mean-Variance):}
\begin{lstlisting}[language=Python]
Q = np.array([[0.1, 0.01], [0.01, 0.2]])
c = np.zeros(2)
A = np.ones((1,2)); b = np.array([1.0])  # sum w=1? Wait, equality.
res = solve_qp(Q, c, np.zeros((0,2)), np.zeros(0), A, b)
print(res['x_opt'])  # ~ [0.55, 0.45]
\end{lstlisting}

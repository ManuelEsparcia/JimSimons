Modulo: simons_core/schemas.py

1) Objetivo
Centralizar esquemas canonicos de datasets y mensajes internos para garantizar consistencia, validacion y trazabilidad en todo el pipeline.

2) Alcance
Incluye definicion de columnas/tipos, llaves primarias logicas, reglas de nulabilidad y validadores de schema.

3) Schemas clave
- prices_raw_schema
- prices_adjusted_schema
- universe_history_schema
- features_schema
- labels_schema
- predictions_schema
- positions/orders/fills schema
- qc/gate/report schema

4) Reglas comunes
- columnas obligatorias por dataset.
- tipos canonicos (date, float64, int64, string, bool).
- PK logica por entidad (ej. date+symbol).
- metadata minima: run_id, ingest_ts_utc, config_hash.

5) Validacion de schema
- validate_required_columns(df, schema)
- validate_dtypes(df, schema)
- validate_primary_key_uniqueness(df, keys)
- validate_nullable_constraints(df, schema)

6) Versionado
- schema_version por dataset.
- migraciones documentadas (v1->v2...).
- compat checks para lectores/escritores.

7) Salidas
- catalogo de schemas y utilidades de validacion.
- reporte de errores de schema detallado.

8) CLI/Test hook
python -m simons_core.schemas --check path_to_dataset --schema features_schema

9) DoD
- todos los datasets core con schema canonico verificable.

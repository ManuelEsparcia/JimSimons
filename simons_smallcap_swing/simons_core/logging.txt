\section{Objetivo Estratégico}
Estandariza logs JSON con \texttt{run\_id} para correlation en pipelines swing (DataProvider $\to$ Predictor $\to$ Execution). Renaissance usa equivalente para root-cause en 66\% returns: Detecta drifts en 1min \cite{simons_returns}.

\section{Alcance y API Pública}
- \texttt{get\_logger(name, run\_context)}}: Factory structlog.
- \texttt{with\_step(logger, "feature\_build")}: Context manager.
- Campos: \texttt{"ts_utc", "level", "run_id", "event", "payload"}.

Reglas: INFO para signals, ERROR con \texttt{error\_code}, no PII.

\section{Matemáticas de Observability}
\subsection{Error Rate y Alertas Predictivas}
Poisson para log events:
\[
\lambda_{err} = \frac{\#ERROR}{T}, \quad \text{alert} \iff \lambda > \mu + 3\sigma
\]
Métricas: $\text{error\_rate} = \mathbb{P}(\text{ERROR}|module)}$.

\subsection{Correlation y Propagate}
Trace ID como Markov chain: $P(state_t | trace_{t-1})$, para depuración causal \cite{mlops_finance}.

\section{Optimizaciones para Rentabilidad Máxima}
- \textbf{Anomaly Detection:} Integra Prophet para $\lambda(t)$ forecast.
- \textbf{Ejemplo Código:}
  \begin{lstlisting}
  from simons_core.logging import get_logger
  logger = get_logger(__name__, run_context)
  with logger.with_step("rebalance"):
      logger.info("event"="positions_updated", payload={"sharpe":2.5})
  \end{lstlisting}

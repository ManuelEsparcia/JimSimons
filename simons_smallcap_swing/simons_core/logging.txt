Modulo: simons_core/logging.py

1) Objetivo
Estandarizar logging estructurado en todo el sistema para trazabilidad por run_id, depuracion y monitoreo operativo.

2) Alcance
Incluye:
- logger factory comun
- formato JSON/text configurable
- campos estandar (run_id, step, module, severity, ts)
- context propagation y correlation ids

3) API sugerida
- get_logger(name, run_context=None)
- with_step(logger, step_name)
- log_event(logger, level, event_name, payload)
- bind_context(logger, **kwargs)

4) Reglas
- logs de negocio en INFO/WARNING/ERROR.
- evitar logs sensibles (credenciales/PII).
- mensajes accionables y con codigos de error.

5) Campos canonicos de log
- timestamp_utc
- level
- module
- run_id
- step
- event
- message
- error_code (nullable)
- extra_payload

6) Integracion
- compatible con ops/monitoring y collectors externos.
- opcion de sink a archivo + stdout.

7) Metricas
- log_count_by_level
- error_rate_by_module
- dropped_log_events

8) CLI/Test hook
python -m simons_core.logging --smoke

9) DoD
- logging uniforme en todos los modulos core.

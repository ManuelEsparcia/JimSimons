Modulo: simons_core/interfaces.py

1) Objetivo
Definir interfaces/contratos comunes del sistema para estandarizar componentes de datos, modelado, portfolio, ejecucion y validacion.

2) Alcance
Incluye interfaces abstractas (protocols/ABCs), tipos canonicos y contratos de entrada/salida entre modulos.

3) Principios
- contratos explicitos y versionados.
- tipado fuerte en boundaries.
- compatibilidad hacia atras controlada.

4) Interfaces nucleares sugeridas
- DataProvider: fetch/read/write/list_versions
- FeatureBuilder: build/validate/persist
- LabelBuilder: build/validate
- ModelTrainer: fit/evaluate/export
- Predictor: load/predict/postprocess
- PortfolioEngine: size/constrain/rebalance
- ExecutionEngine: simulate_fills/costs
- Validator: run_gate/summary

5) Tipos canonicos
- RunContext (run_id, config_hash, asof_date, seed)
- ArtifactRef (path, version, schema_version, checksum)
- GateResult (status, score, reasons, metrics)
- StepResult (status, outputs, duration, errors)

6) Reglas de versionado
- cada interface con version semantica.
- cambios breaking requieren bump major.
- adapters para compatibilidad transitoria.

7) Validaciones
- implementaciones deben cumplir firmas y tipos.
- smoke tests por interface.
- contract tests de I/O minimos.

8) Salidas
- interfaces y tipos reutilizables por todo el repo.
- metadata de compatibilidad entre versiones.

9) CLI/Test hook sugerido
python -m simons_core.interfaces --validate-contracts

10) DoD
- contratos centrales definidos, documentados y testeables.

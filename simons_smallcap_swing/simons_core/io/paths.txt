Modulo: simons_core/io/paths.py

1) Objetivo
Gestionar rutas canonicas de proyecto (data/models/reports/ops) de forma portable, versionada y consistente entre modulos.

2) Alcance
Incluye resolucion de roots, builders de path por dataset/run, convenciones de particionado y utilidades de seguridad de path.

3) Convenciones base
- root por entorno: local/dev/prod.
- particionado preferente por date/run_id.
- nombres deterministas y sin espacios.

4) API sugerida
- get_project_root()
- build_data_path(domain, entity, date=None, run_id=None, partition=None)
- build_model_artifact_path(model_id, run_id, version=None)
- build_report_path(scope, run_id)
- ensure_parent_dirs(path)

5) Reglas
- no hardcode de rutas absolutas en modulos de negocio.
- toda escritura pasa por path builder.
- validar que path final quede dentro de root permitido.

6) Salidas
- rutas estandarizadas para lectura/escritura.
- metadata de path conventions en manifest.

7) CLI/Test hook
python -m simons_core.io.paths --print-path data price raw --date 2026-03-05

8) DoD
- convencion unica de rutas aplicada en todo el repo.

Modulo: simons_core/io/parquet_store.py

1) Objetivo
Proveer utilidades comunes para lectura/escritura de Parquet con schema checks, particionado consistente y metadata de reproducibilidad.

2) Alcance
Incluye:
- write_parquet(versioned snapshot)
- read_parquet(range/columns/filters)
- append/overwrite controlado
- validacion de schema al escribir/leer
- manifest sidecar por run

3) Entradas comunes
- dataframe
- destination_path
- schema_ref
- partition_cols
- write_mode
- run_context

4) Salidas
- archivos parquet particionados
- manifest json con row_count/schema_hash/checksum

5) Reglas de escritura
- validar schema antes de persistir.
- dedupe PK logica opcional previo a write.
- modos permitidos: fail_if_exists, overwrite_partition, append_versioned.

6) Reglas de lectura
- column projection obligatoria cuando posible.
- filtros pushdown por fecha/symbol.
- validacion opcional de schema esperada.

7) Manejo de errores
- schema mismatch -> fail descriptivo.
- particion corrupta -> marcar y aislar.
- path no permitido -> reject.

8) Observabilidad
- rows_written/read
- bytes_written/read
- write/read_latency_sec

9) CLI/Test hook
python -m simons_core.io.parquet_store --validate path_to_dataset --schema features_schema

10) DoD
- capa parquet reusable con checks robustos e idempotencia.

Modulo: validation/multiple_testing.py

1) Objetivo
Controlar riesgo de falsos positivos por multiple testing en research cuantitativo, ajustando significancia y performance aparente para evitar data snooping.

2) Alcance
Incluye:
- Correcciones de p-values (Bonferroni, Holm, Benjamini-Hochberg).
- Control FWER/FDR.
- Metricas ajustadas tipo Deflated Sharpe Ratio (DSR) cuando aplique.
- Reporte de hipotesis que sobreviven ajuste.
- Gate de promocion estadistica.

No incluye:
- Generacion de hipotesis/modelos.
- Validacion final integrada (la hace validation_suite).

3) Dependencias
Upstream:
- resultados de experimentos (metricas, p-values, n_trials)
- informacion de dependencia entre pruebas (opcional)
- configuracion estadistica

Downstream:
- validation/validation_suite
- research/experiment_tracker

4) Contrato de entrada
Entradas minimas:
- experiment_results_path
- multiple_testing_config_path
- run_id

Campos minimos:
- experiment_id
- metric_primary
- p_value_raw (si disponible)
- sample_size
- strategy_family (opcional)

Parametros clave:
- correction_method: bonferroni | holm | bh_fdr
- alpha_level
- use_deflated_sharpe (bool)
- dsr_params
- min_effect_size

5) Contrato de salida
- validation/runs/{run_id}/multiple_testing_summary.json
- validation/runs/{run_id}/hypothesis_table.parquet
- validation/runs/{run_id}/survivors.parquet
- validation/runs/{run_id}/manifest.json

Campos sugeridos hypothesis_table:
- experiment_id
- p_value_raw
- p_value_adj
- reject_null_raw
- reject_null_adj
- effect_size
- dsr (opcional)
- multiple_testing_gate

6) Metodologia recomendada
1. Cargar resultados y limpiar duplicados.
2. Aplicar correccion seleccionada sobre p-values.
3. Calcular decision de rechazo ajustado.
4. Opcional: calcular DSR u otra metrica deflactada.
5. Filtrar survivors por significancia + effect size minimo.
6. Emitir score/gate global.

7) Gates sugeridos
- Gate 1: n_survivors_adj >= minimo.
- Gate 2: p_value_adj de candidatos principales < alpha.
- Gate 3: effect_size >= umbral minimo.
- Gate 4 (opcional): DSR >= umbral.

Resultado:
- pass/warn/fail segun umbrales configurados.

8) Reglas PIT y anti-leakage
- Solo usar resultados OOS o validacion temporal correcta.
- No mezclar metricas IS puras como evidencia final de significancia.
- Contabilizar numero real de trials ejecutados (incluye descartes).

9) Validaciones minimas
- p-values en [0,1].
- experiment_id unicos o deduplicados.
- sample_size positivo.
- coherencia entre metricas y signo de effect_size.

10) Manejo de casos especiales
- p-values ausentes:
  - estimar via bootstrap/permutation si config lo permite.
- alta dependencia entre pruebas:
  - preferir BH-FDR o metodos robustos.
- pocos experimentos:
  - advertir baja potencia estadistica.

11) Observabilidad
Logging:
- INFO: metodo aplicado y alpha.
- WARNING: tasa alta de rechazos raw no sostenidos tras ajuste.
- ERROR: datos invalidos para inferencia.

Metricas:
- n_experiments
- n_survivors_raw
- n_survivors_adj
- fdr_estimate
- runtime_sec

12) CLI sugerida
python -m validation.multiple_testing ^
  --experiment-results-path research/experiment_tracker/results_latest.parquet ^
  --config-path configs/validation/multiple_testing.yaml ^
  --run-id multiple_testing_20260305_001

13) Estructura interna recomendada
- load_multiple_testing_config(...)
- load_experiment_results(...)
- apply_pvalue_correction(...)
- compute_deflated_sharpe_if_enabled(...)
- evaluate_survivors(...)
- compute_multiple_testing_gate(...)
- persist_multiple_testing_outputs(...)
- main()

14) Criterios de aceptacion
- Ajuste multiple testing reproducible y documentado.
- Lista de survivors con criterio estadistico claro.
- Gate utilizable por validation_suite.

15) Casos de prueba minimos
- correccion Bonferroni sobre ejemplo sintetico.
- BH-FDR con ranking correcto.
- DSR calculado cuando se activa.
- fail controlado por datos invalidos.

16) Checklist rapido
- [ ] Correcciones p-value implementadas.
- [ ] Filtro por effect size.
- [ ] DSR opcional.
- [ ] Gate pass/warn/fail.
- [ ] Reportes summary + survivors.

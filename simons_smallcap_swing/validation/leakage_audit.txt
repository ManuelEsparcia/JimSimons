Modulo: validation/leakage_audit.py

1) Objetivo
Detectar, cuantificar y bloquear cualquier fuga de informacion (look-ahead/data leakage) en datos, features, labels, splits y evaluacion de modelos.

2) Alcance
Incluye:
- Auditoria temporal de timestamps y asof dates.
- Auditoria de joins (future backfill, accidental forward fill).
- Verificacion de purged splits y embargo.
- Pruebas proxy de leakage en performance/predicciones.
- Emision de hallazgos con severidad y gate final.

No incluye:
- Correccion automatica de datasets.
- Redefinicion de estrategia/modelo.

3) Dependencias
Upstream:
- features/store + metadata/lineage
- labels/store + metadata
- labels/splits/purged_splits
- predicciones OOF/OOS
- universe/history y precios PIT

Downstream:
- validation/validation_suite
- bloqueo de entrenamiento/backtest si fail critico

4) Contrato de entrada
Entradas minimas:
- features_path
- labels_path
- splits_path
- predictions_path (opcional pero recomendado)
- leakage_config_path
- run_id

5) Contrato de salida
- validation/runs/{run_id}/leakage_findings.parquet
- validation/runs/{run_id}/leakage_summary.json
- validation/runs/{run_id}/leakage_gate.txt
- validation/runs/{run_id}/leakage_manifest.json

Campos recomendados en findings:
- check_id
- layer (data/features/labels/splits/model_eval)
- severity (low/medium/high/critical)
- status (pass/warn/fail)
- metric_value
- threshold
- description
- affected_columns

6) Checks obligatorios
Data/feature timestamp checks:
- source_available_ts <= decision_cut_ts.
- no filas con asof posterior a fecha de decision.

Join checks:
- no forward merge beyond allowed horizon.
- no use of universe_current in historical rows.

Label checks:
- forward window correcta y sin uso de precios previos mal alineados.
- decision_lag coherente con features.

Split checks:
- no overlap de event intervals entre train y test tras purge.
- embargo aplicado.

Model proxy checks:
- performance implausiblemente alta en train/valid con deterioro extremo OOS.
- shift tests (feature_t vs target_t+1/t-1) para senales sospechosas.

7) Clasificacion de severidad
- low: inconsistencia menor, no bloqueante.
- medium: riesgo moderado, requiere review.
- high: riesgo alto, gate en warn/fail segun conteo.
- critical: fuga confirmada; gate fail inmediato.

8) Regla de gate
- pass: 0 hallazgos high/critical y low/medium bajo umbral.
- warn: sin critical, pero high limitrofes o varios medium.
- fail: >=1 critical o multiples high sobre umbral.

9) Flujo del proceso
1. Cargar config y artefactos.
2. Ejecutar checks por capa.
3. Consolidar findings y severidades.
4. Calcular leakage_risk_score.
5. Emitir gate y persistir reportes.

10) Validaciones minimas
- consistencia de run_id entre artefactos.
- cobertura minima de fechas revisadas.
- checks criticos ejecutados (no skip silencioso).

11) Observabilidad
Logging:
- INFO: progreso por grupo de checks.
- WARNING: hallazgos medium/high.
- ERROR: leakage critical confirmado.

Metricas:
- n_checks_total
- n_checks_failed
- n_findings_by_severity
- leakage_risk_score
- runtime_sec

12) CLI sugerida
python -m validation.leakage_audit ^
  --features-path features/store ^
  --labels-path labels/store ^
  --splits-path labels/splits/purged_splits_latest.parquet ^
  --predictions-path models/predictions/lgbm/oof.parquet ^
  --config-path configs/validation/leakage.yaml ^
  --run-id leakage_audit_20260305_001

13) Estructura interna recomendada
- load_leakage_config(...)
- run_timestamp_checks(...)
- run_join_checks(...)
- run_label_alignment_checks(...)
- run_split_overlap_checks(...)
- run_model_proxy_checks(...)
- classify_findings(...)
- compute_leakage_gate(...)
- persist_leakage_outputs(...)
- main()

14) Criterios de aceptacion
- Ejecuta checks criticos en todas las capas.
- Entrega findings accionables con severidad.
- Emite gate pass/warn/fail reproducible.

15) Casos de prueba minimos
- asof future en features dispara critical.
- split con overlap train/test dispara fail.
- mismatch de lag features-labels detectado.
- dataset limpio pasa sin high/critical.

16) Checklist rapido
- [ ] Checks de timestamps/asof.
- [ ] Checks de joins y universe PIT.
- [ ] Checks labels/splits.
- [ ] Proxy checks de leakage en model outputs.
- [ ] Gate final y reportes.

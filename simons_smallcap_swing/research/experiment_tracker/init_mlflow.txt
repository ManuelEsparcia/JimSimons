Modulo: research/experiment_tracker/init_mlflow.py

1) Objetivo
Inicializar y estandarizar el backend de experiment tracking (MLflow u equivalente), definiendo convenciones de proyectos/runs/tags para trazabilidad total de research.

2) Alcance
Incluye:
- setup de tracking URI y artifact store.
- creacion/validacion de experiment namespaces.
- convenciones de tags y metadata obligatoria.
- helpers para iniciar/cerrar runs con contexto.

No incluye:
- logging detallado de artefactos (artifact_logger).
- orquestacion de sweeps (sweep_launcher).

3) Dependencias
Upstream:
- configuracion de entorno (local/remote tracking)
- credenciales y storage backend
- convenciones globales de naming del proyecto

Downstream:
- research/experiment_tracker/artifact_logger.py
- research/experiment_tracker/sweep_launcher.py
- research/research_pipeline.py

4) Contrato de entrada
Entradas minimas:
- mlflow_config_path
- project_name
- run_context (opcional)

Parametros clave:
- tracking_uri
- artifact_uri
- experiment_name_pattern
- required_tags
- allow_auto_create_experiment

5) Contrato de salida
- experiment_tracker/init/{run_id}/init_summary.json
- experiment_tracker/init/{run_id}/manifest.json

Efectos esperados:
- experiment creado/validado.
- entorno de tracking listo para runs.

6) Tags/metadata obligatorios sugeridos
- run_id
- git_commit
- config_hash
- data_snapshot_hash
- schema_version
- owner
- stage (research/staging/prod_candidate)

7) Flujo del proceso
1. Cargar config y resolver tracking URI.
2. Validar conectividad y permisos.
3. Crear o resolver experiment por nombre.
4. Validar tags obligatorios.
5. Emitir resumen y estado ready.

8) Validaciones minimas
- tracking URI accesible.
- artifact store escribible.
- experiment_id resoluble.
- tags obligatorios presentes en run bootstrap.

9) Manejo de errores
- fallo de conexion -> fail inmediato.
- experimento inexistente y auto_create=false -> fail.
- permisos insuficientes -> fail con codigo claro.

10) Observabilidad
Logging:
- INFO: tracking URI y experiment resuelto.
- WARNING: uso de defaults no recomendados.
- ERROR: init fallido por permisos/conexion.

Metricas:
- init_success_flag
- init_latency_sec
- experiment_exists_flag

11) CLI sugerida
python -m research.experiment_tracker.init_mlflow ^
  --config-path configs/research/mlflow.yaml ^
  --project-name simons_smallcap_swing ^
  --run-id init_mlflow_20260305_001

12) Estructura interna recomendada
- load_mlflow_config(...)
- validate_tracking_backend(...)
- resolve_or_create_experiment(...)
- validate_required_tags(...)
- build_init_summary(...)
- persist_init_outputs(...)
- main()

13) Criterios de aceptacion
- backend tracking listo y validado.
- convenciones de metadata aplicadas.
- salida reproducible con estado clear.

14) Casos de prueba minimos
- init exitoso en entorno local.
- fallo controlado por tracking URI invalido.
- experimento existente se reusa correctamente.
- missing required tag dispara fail.

15) Checklist rapido
- [ ] resolver tracking/artifact URI.
- [ ] crear/validar experiment.
- [ ] validar metadata obligatoria.
- [ ] persistir summary/manifest.

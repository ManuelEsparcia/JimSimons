\section{Introducción Teórica y Objetivo}
\texttt{sweep\_launcher.py} orquesta hyperparameter sweeps para ML en swing trading, minimizando data leakage via trazabilidad jerárquica (parent/child runs en MLflow). Objetivo: \(\max_\theta \metric(\theta)\) s.t. budget \( B \), donde \(\metric = \OOS_\IC\) (Information Coefficient out-of-sample) \parencite{lopez2020}.

\textbf{Estrategias}: Grid \(\Theta_\grid\), Random \(\Theta_\unif\), Bayes \(\acq(\x | D) = \EI(f(\x))\) con GP surrogate:
\[
f(\x) \sim \GP(\mu(\x), k(\x,\x')), \quad k = \Matérn(\nu=5/2, \ell),
\]
\(\EI(\x) = \E[\max(0, f(\x) - f(\x^+))] \) para explotación/exploración \parencite{jones1998}.

Libros clave: \emph{``Experimentation in Software Engineering''} Wohlin (2012); \emph{``Bayesian Optimization in Action''} Frazier (2018).

\section{Contrato Matemático de Entrada/Salida}
Entradas: \( \mathbf{I} = (\confpath, \expname, \objmetric, \runid) \), space \(\Theta \subset \R^d\), \( d \leq 20 \).

Reglas: \( N_\trials \leq 1000 \), parallel \( P \leq 32 \), seed para \(\repro\).

\section{Governance y Flujo}
Governance: Registrar fails, fix \(\obj\) ex-ante, \( N_\trials \) para FDR control.

Flujo:
\begin{enumerate}
\item Parent run \( r_\parent = \init(\exp) \).
\item Proposals: \(\x_k \sim \pi(\cdot | D_{k-1})\), \(\pi =\bayes\).
\item Trials child: \( r_{\child_k} \leftarrow \launch(\x_k, timeout) \).
\item Early stop: \(\patience=10\), \(\delta_\metric < \epsilon\).
\item Leaderboard: \(\sort(\{ (\metric_i, \x_i) \})\).
\end{enumerate}

Validaciones: \(\Theta\) bounded, no NaN \(\metric\).

\section{Validaciones, Errores y Observabilidad}
Errores: Retry \( \tau_k = \min(2^k, 300)s \), kill SIGTERM on timeout.

Métricas:
\[
\frac{n_\success}{n_\total}, \quad \regret = \max_\history \metric - \bar{\metric}.
\]

CLI con Hydra para config composition.

\mu_{k}(\x) = k_\star (K + \sigma^2 I)^{-1} \y, \quad \sigma^2(\x) = k(\x,\x) - k_\star (K + \sigma^2 I)^{-1} k_\star^T.
\]

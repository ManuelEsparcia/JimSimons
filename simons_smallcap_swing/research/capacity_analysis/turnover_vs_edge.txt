\title{turnover\_vs\_edge.py --- Net Edge as a Function of Turnover}
\author{}
\date{}
\maketitle

\section{Purpose}

The purpose of \texttt{turnover\_vs\_edge.py} is to estimate how much of the strategy's gross edge survives after turnover-dependent execution costs and signal-decay effects are taken into account. The module is not an optimizer in the abstract; it is a diagnostic and decision-support tool for determining whether the current rebalance intensity is economically justified.

Operationally, the module must answer four questions:

\begin{enumerate}[label=\arabic*.]
    \item How does net edge change as realized turnover increases?
    \item At what turnover level does additional trading cease to improve net performance?
    \item How much gross alpha is being lost when turnover is throttled?
    \item What turnover range is admissible once costs, volatility, and implementation constraints are included?
\end{enumerate}

The module therefore produces a turnover-response curve
\[
T \mapsto \left(\mu_{\mathrm{gross}}(T),\; \mu_{\mathrm{net}}(T),\; SR_{\mathrm{net}}(T)\right),
\]
together with an admissible turnover region and a recommended operating point.

\section{Module Contract}

\subsection{Required Inputs}

The module requires point-in-time safe daily data after signal generation and before final execution:

\begin{itemize}
    \item \texttt{rebalance\_panel}: one row per \((date, asset)\), containing
    \[
    \{\texttt{date}, \texttt{asset}, \texttt{w\_prev}, \texttt{w\_target}, \texttt{forecast\_alpha\_bps},
    \texttt{volatility}, \texttt{adv\_dollar}, \texttt{spread\_bps}\}.
    \]
    \item \texttt{impact\_params}: calibrated cost parameters by liquidity / volatility bucket, e.g.
    \[
    \{\texttt{impact\_Y}, \texttt{impact\_delta}, \texttt{impact\_gamma}, \texttt{participation\_cap}\}.
    \]
    \item \texttt{realized\_returns} or an equivalent forward-return panel for evaluating realized alpha retention under throttled rebalancing.
    \item \texttt{config}: turnover grid or throttle grid, annualization constants, evaluation horizon, and minimum acceptable net Sharpe.
\end{itemize}

\subsection{Outputs}

For each grid point \(g\), corresponding either to a turnover target or to a rebalance-throttle parameter, the module produces

\[
\{\texttt{grid\_value}, \texttt{turnover\_annualized}, \texttt{gross\_edge\_bps},
\texttt{cost\_drag\_bps}, \texttt{net\_edge\_bps}, \texttt{gross\_return},
\texttt{net\_return}, \texttt{gross\_sharpe}, \texttt{net\_sharpe},
\texttt{participation\_mean}, \texttt{capacity\_stress}, \texttt{status}\}.
\]

Results are written to a point-in-time safe artifact such as

\[
\texttt{research/capacity\_analysis/artifacts/turnover\_vs\_edge.parquet}.
\]

\section{Mathematical Framework}

\subsection{Realized Turnover}

Let \(w_{i,t-1}\) be the live post-trade portfolio weight carried into date \(t\), and let \(w^{\star}_{i,t}\) be the desired target weight generated by the portfolio construction layer. Define the unthrottled desired trade as

\begin{equation}
\Delta w^{\star}_{i,t} = w^{\star}_{i,t} - w_{i,t-1}.
\end{equation}

Rather than writing down an arbitrary mapping from signal strength to turnover, the module parameterizes execution intensity through a throttle \(\lambda \in [0,1]\):

\begin{equation}
w^{(\lambda)}_{i,t} = w_{i,t-1} + \lambda \,\Delta w^{\star}_{i,t}.
\end{equation}

Here,
\begin{itemize}
    \item \(\lambda = 1\) means full rebalance,
    \item \(\lambda = 0\) means no rebalance,
    \item intermediate values represent partial trading toward the target.
\end{itemize}

The one-period turnover induced by \(\lambda\) is

\begin{equation}
T_t(\lambda) = \frac{1}{2}\sum_i \left| w^{(\lambda)}_{i,t} - w_{i,t-1} \right|
= \frac{\lambda}{2}\sum_i |\Delta w^{\star}_{i,t}|.
\end{equation}

Annualized turnover is then

\begin{equation}
T_{\mathrm{ann}}(\lambda) = \frac{252}{N_t}\sum_{t=1}^{N_t} T_t(\lambda),
\end{equation}

or an equivalent business-day annualization consistent with the rest of the backtest stack.

\subsection{Gross Edge Retention Under Throttling}

Lower turnover reduces costs, but it can also reduce alpha capture because the portfolio moves more slowly toward the desired target. Therefore the relevant object is not cost alone but the net effect of slower adjustment.

Let \(r^{\mathrm{gross}}_t(\lambda)\) denote the gross portfolio return generated when positions are updated using throttle \(\lambda\). Then the annualized gross edge is

\begin{equation}
\mu_{\mathrm{gross}}(\lambda) = \mathrm{AnnMean}\big(r^{\mathrm{gross}}_t(\lambda)\big).
\end{equation}

There is no need to impose an arbitrary closed-form expression for gross-edge decay. It should be measured directly from the throttled backtest or from a realistic simulation of delayed target convergence.

\subsection{Cost Model}

For each asset \(i\) and date \(t\), let traded dollar volume under throttle \(\lambda\) be \(V_{i,t}(\lambda)\). Per-dollar trading cost is decomposed as

\begin{equation}
c_{i,t}(\lambda) =
c^{\mathrm{spread}}_{i,t}(\lambda)
+ c^{\mathrm{temp}}_{i,t}(\lambda)
+ c^{\mathrm{perm}}_{i,t}(\lambda),
\end{equation}

with total daily cost

\begin{equation}
K_t(\lambda) = \sum_i c_{i,t}(\lambda)\,V_{i,t}(\lambda).
\end{equation}

A parsimonious implementation is:

\begin{equation}
c^{\mathrm{spread}}_{i,t}(\lambda) = \frac{1}{2}\,\mathrm{spread}_{i,t},
\end{equation}

\begin{equation}
c^{\mathrm{temp}}_{i,t}(\lambda)
=
Y_{i,t}\,\sigma_{i,t}\left(
\frac{V_{i,t}(\lambda)}{ADV_{i,t} + \varepsilon}
\right)^{\delta_{i,t}},
\end{equation}

\begin{equation}
c^{\mathrm{perm}}_{i,t}(\lambda) = \gamma_{i,t}\,V_{i,t}(\lambda).
\end{equation}

The important point is that the coefficients \((Y,\delta,\gamma)\) must come from the execution / impact model. They are not decorative constants and should not be hard-coded as universal values.

\subsection{Net Edge and Net Sharpe}

Define daily net return under throttle \(\lambda\) as

\begin{equation}
r^{\mathrm{net}}_t(\lambda)
=
r^{\mathrm{gross}}_t(\lambda)
-
\frac{K_t(\lambda)}{A_t}
-
f_t(\lambda),
\end{equation}

where \(A_t\) is capital and \(f_t(\lambda)\) includes borrow, financing, or other non-trading drags if relevant.

The annualized net edge is

\begin{equation}
\mu_{\mathrm{net}}(\lambda) = \mathrm{AnnMean}\big(r^{\mathrm{net}}_t(\lambda)\big),
\end{equation}

the annualized net volatility is

\begin{equation}
\sigma_{\mathrm{net}}(\lambda) = \mathrm{AnnStd}\big(r^{\mathrm{net}}_t(\lambda)\big),
\end{equation}

and the net Sharpe is

\begin{equation}
SR_{\mathrm{net}}(\lambda)
=
\frac{\mu_{\mathrm{net}}(\lambda) - r_f}
{\sigma_{\mathrm{net}}(\lambda) + \varepsilon}.
\end{equation}

\section{Optimization Objective}

The module should not pretend there is a unique universal optimum. It reports several economically meaningful operating points.

\subsection{Net-Edge Maximizer}

\begin{equation}
\lambda^{\star}_{\mu}
\in
\arg\max_{\lambda \in [0,1]} \mu_{\mathrm{net}}(\lambda).
\end{equation}

\subsection{Net-Sharpe Maximizer}

\begin{equation}
\lambda^{\star}_{SR}
\in
\arg\max_{\lambda \in [0,1]} SR_{\mathrm{net}}(\lambda).
\end{equation}

\subsection{Admissible Region}

Given a minimum acceptable net Sharpe \(SR_{\min}\) and a maximum acceptable average participation \(\pi_{\max}\), define the admissible set

\begin{equation}
\Lambda_{\mathrm{adm}}
=
\left\{
\lambda \in [0,1] :
SR_{\mathrm{net}}(\lambda) \ge SR_{\min},
\;
\bar{\pi}(\lambda) \le \pi_{\max}
\right\},
\end{equation}

where \(\bar{\pi}(\lambda)\) is the average participation rate induced by \(\lambda\).

In practice, the recommended operating point should come from this admissible region rather than from unconstrained maximization.

\section{Implementation Logic}

\begin{lstlisting}
import numpy as np
import pandas as pd

EPS = 1e-12

def annualized_mean(x, periods_per_year=252):
    return periods_per_year * np.nanmean(x)

def annualized_std(x, periods_per_year=252):
    return np.sqrt(periods_per_year) * np.nanstd(x, ddof=1)

def throttle_targets(df, lam):
    out = df.copy()
    out["trade_weight"] = lam * (out["w_target"] - out["w_prev"])
    out["w_exec"] = out["w_prev"] + out["trade_weight"]
    return out

def turnover_from_trade_weight(trade_weight):
    return 0.5 * np.abs(trade_weight).sum()

def impact_cost_per_dollar(volume, adv, sigma, Y, delta):
    part = np.maximum(volume / (adv + EPS), 0.0)
    return Y * sigma * np.power(part, delta)

def permanent_cost_per_dollar(volume, gamma):
    return gamma * volume

def evaluate_lambda(day_df, lam, capital):
    df = throttle_targets(day_df, lam)

    df["trade_dollar"] = np.abs(df["trade_weight"]) * capital
    df["participation"] = df["trade_dollar"] / (df["adv_dollar"] + EPS)

    df["spread_cost_per_dollar"] = 0.5 * df["spread_bps"] * 1e-4
    df["temp_cost_per_dollar"] = impact_cost_per_dollar(
        volume=df["trade_dollar"].to_numpy(),
        adv=df["adv_dollar"].to_numpy(),
        sigma=df["volatility"].to_numpy(),
        Y=df["impact_Y"].to_numpy(),
        delta=df["impact_delta"].to_numpy(),
    )
    df["perm_cost_per_dollar"] = permanent_cost_per_dollar(
        volume=df["trade_dollar"].to_numpy(),
        gamma=df["impact_gamma"].to_numpy(),
    )

    df["cost_dollar"] = (
        df["spread_cost_per_dollar"]
        + df["temp_cost_per_dollar"]
        + df["perm_cost_per_dollar"]
    ) * df["trade_dollar"]

    # Gross alpha capture should come from the throttled portfolio path,
    # not from a hard-coded analytic shortcut.
    gross_pnl = df["gross_alpha_dollar_throttled"].sum()
    cost_pnl = df["cost_dollar"].sum()

    return {
        "turnover_daily": turnover_from_trade_weight(df["trade_weight"]),
        "gross_return_daily": gross_pnl / capital,
        "net_return_daily": (gross_pnl - cost_pnl) / capital,
        "participation_mean_daily": df["participation"].mean(),
        "capacity_stress_daily": (df["participation"] > df["participation_cap"]).mean(),
    }

panel = pd.read_parquet(rebalance_panel_path)

lambdas = np.linspace(0.0, 1.0, 21)
results = []

for lam in lambdas:
    daily_rows = []
    for dt, day_df in panel.groupby("date"):
        daily_rows.append(evaluate_lambda(day_df, lam=lam, capital=capital))
    daily = pd.DataFrame(daily_rows)

    gross_return = annualized_mean(daily["gross_return_daily"])
    net_return = annualized_mean(daily["net_return_daily"])
    gross_vol = annualized_std(daily["gross_return_daily"])
    net_vol = annualized_std(daily["net_return_daily"])

    results.append({
        "grid_value": lam,
        "turnover_annualized": 252 * daily["turnover_daily"].mean(),
        "gross_edge_bps": 1e4 * gross_return,
        "cost_drag_bps": 1e4 * (gross_return - net_return),
        "net_edge_bps": 1e4 * net_return,
        "gross_return": gross_return,
        "net_return": net_return,
        "gross_sharpe": gross_return / (gross_vol + EPS),
        "net_sharpe": net_return / (net_vol + EPS),
        "participation_mean": daily["participation_mean_daily"].mean(),
        "capacity_stress": daily["capacity_stress_daily"].mean(),
    })

curve = pd.DataFrame(results).sort_values("turnover_annualized")

sr_min = 1.0
pi_max = 0.05

curve["status"] = "inadmissible"
mask = (
    (curve["net_sharpe"] >= sr_min) &
    (curve["participation_mean"] <= pi_max)
)
curve.loc[mask, "status"] = "admissible"

curve.to_parquet(output_path, index=False)
\end{lstlisting}

\section{Calibration Requirements}

This module is only as good as its calibration. The following requirements are mandatory.

\subsection{Cost Calibration}

Spread, temporary impact, and permanent impact must be imported from the execution-cost framework or estimated from fills / TCA. Constants such as
\[
\gamma T + \eta \sqrt{T}
\]
may be useful for toy intuition, but they are not sufficient as a production specification unless linked back to actual traded notional, liquidity, and fills.

\subsection{Alpha-Retention Measurement}

The key quantity missing in simplistic turnover models is the amount of gross alpha retained under slower trading. This must be estimated from one of the following:

\begin{enumerate}[label=\arabic*.]
    \item a true throttled backtest in which targets are only partially implemented,
    \item a delayed-execution simulator with signal decay,
    \item a validated reduced-form model mapping trade delay to alpha loss.
\end{enumerate}

Without explicit alpha-retention measurement, the module understates the cost of reducing turnover.

\subsection{Participation Constraints}

If throttling is used partly to control market footprint, the module must report not only net edge but also average participation, tail participation, and the fraction of names whose execution exceeds policy limits.

\section{Validation Protocol}

A turnover policy should be accepted only if it survives the following out-of-sample checks:

\begin{enumerate}[label=\arabic*.]
    \item \textbf{Monotonic cost realism}: cost drag rises with realized turnover in a way consistent with fills or TCA.
    \item \textbf{Alpha-retention realism}: low-turnover variants lose gross edge in a way consistent with known signal decay.
    \item \textbf{Economic optimum stability}: the recommended turnover range is reasonably stable across subperiods and liquidity regimes.
    \item \textbf{Incrementality}: the module adds useful information beyond raw turnover alone by explaining the interaction between turnover, costs, and realized edge.
    \item \textbf{Deployability}: the chosen operating region respects participation and execution constraints.
\end{enumerate}

Useful diagnostics include:
\begin{itemize}
    \item turnover deciles versus realized implementation shortfall,
    \item turnover versus net edge curve with confidence bands,
    \item turnover versus average and 95th-percentile participation,
    \item regime-by-regime optimal turnover ranges,
    \item sensitivity of \(\lambda^{\star}_{\mu}\) and \(\lambda^{\star}_{SR}\) to cost calibration.
\end{itemize}

\section{Integration With the Rest of the System}

This module must integrate explicitly with the following components:

\begin{itemize}
    \item \textbf{rebalance}: source of target weights and destination for turnover throttle recommendations.
    \item \textbf{execution / impact\_model}: source of cost coefficients and participation limits.
    \item \textbf{profitability\_curve}: complementary view of how turnover interacts with deployed capital.
    \item \textbf{crowding\_detector}: optional conditioning variable for reducing turnover in crowded states.
    \item \textbf{portfolio}: consumer of the recommended admissible turnover band.
    \item \textbf{validation}: verifies that turnover adjustments improve realized net performance out-of-sample.
\end{itemize}

\section{Edge Cases and Failure Modes}

The implementation must handle the following cases explicitly:

\begin{enumerate}[label=\arabic*.]
    \item \textbf{Near-zero trading days}: turnover and cost ratios become noisy; use robust aggregation and avoid over-interpreting sparse rebalance dates.
    \item \textbf{Missing liquidity fields}: ADV or spread gaps must be imputed conservatively or excluded from calibration.
    \item \textbf{Highly concentrated books}: turnover reduction may protect costs but increase concentration risk; this should be checked jointly with crowding diagnostics.
    \item \textbf{Regime shifts}: calm-period optima may not survive stressed liquidity conditions; scenario overlays should be run.
    \item \textbf{Signal half-life mismatch}: if signal decay is faster than the rebalance cadence, low-turnover settings may look artificially attractive in cost-only models.
\end{enumerate}

\section{What This Module Is Not}

This module should not contain unsupported claims such as specific Sharpe improvements, return uplifts, or ``optimal zones'' presented as universal empirical truths. Those belong to downstream research results.

The responsibility of \texttt{turnover\_vs\_edge.py} is narrower and more important: to estimate the trade-off between trading intensity and retained net edge under assumptions that are explicit, calibratable, and falsifiable.

\section{Extensions}

Once the core version is validated, the following extensions are legitimate:

\begin{enumerate}[label=\arabic*.]
    \item state-dependent throttling \(\lambda_t = \lambda(s_t)\) conditioned on volatility, crowding, and liquidity,
    \item joint optimization of turnover and capacity constraints,
    \item uncertainty bands via block bootstrap,
    \item regime-conditional turnover surfaces,
    \item dynamic control formulations inspired by the dynamic-trading literature.
\end{enumerate}

These are extensions, not substitutes for a clean core specification.

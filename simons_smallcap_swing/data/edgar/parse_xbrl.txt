Modulo: data/edgar/parse_xbrl.py

1) Objetivo
Normalizar y transformar facts XBRL (submissions/companyfacts) en tabla fundamental canonica lista para construccion PIT.

2) Alcance
Incluye mapping de tags a metricas internas, normalizacion de unidades/escala, seleccion de periodos y dedupe de versiones.

3) Entradas
- submissions_raw_path
- companyfacts_raw_path
- xbrl_mapping_config_path
- run_id

4) Salidas
- data/edgar/parsed/facts_canonical_{run_id}.parquet
- data/edgar/parsed/facts_rejected_{run_id}.parquet
- data/edgar/parsed/manifest_{run_id}.json

5) Campos minimos
- cik
- symbol (si mapping disponible)
- metric_name
- value
- period_end
- period_type (Q/YTTM)
- filed_date
- acceptance_datetime
- source_tag
- unit
- run_id

6) Reglas
- mapping tag->metric_name versionado
- conversion de unidades a base definida
- seleccionar registro mas reciente por (metric,period_end,filed_date)
- no mezclar taxonomias incompatibles sin flag

7) Flujo
1. Cargar raw facts y mapping config.
2. Aplicar mapping y conversion de unidades.
3. Resolver conflictos/duplicados.
4. Emitir canonical + rejected + manifest.

8) Validaciones
- metric_name no nulo para registros aceptados
- value en rango razonable segun metrica (flags)
- fechas coherentes (filed>=period_end)

9) CLI
python -m data.edgar.parse_xbrl ^
  --submissions-path data/edgar/raw/submissions ^
  --companyfacts-path data/edgar/raw/companyfacts ^
  --config-path configs/data/edgar_parse_xbrl.yaml ^
  --run-id parse_xbrl_20260305_001

10) DoD
- tabla canonical consistente para point_in_time.

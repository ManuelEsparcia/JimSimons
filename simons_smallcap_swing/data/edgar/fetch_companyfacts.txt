Modulo: data/edgar/fetch_companyfacts.py

1) Objetivo
Descargar companyfacts de EDGAR por CIK para construir series fundamentales PIT (US-GAAP facts) con versionado reproducible.

2) Alcance
Incluye ingestion de JSON companyfacts, parseo de taxonomias/unidades, persistencia raw y tracking de cobertura.

3) Entradas
- cik_list_path
- taxonomy_filter (opcional)
- config_path
- run_id

4) Salidas
- data/edgar/raw/companyfacts/date={asof}/part-*.parquet
- data/edgar/raw/companyfacts_failures_{run_id}.parquet
- data/edgar/raw/companyfacts_manifest_{run_id}.json

5) Campos minimos
- cik
- taxonomy
- tag
- unit
- end_date
- filed_date
- value
- accession_number (si disponible)
- run_id

6) Checks
- value numerico parseable
- filed_date valida
- unit coherente por tag
- dedupe por (cik,tag,unit,end_date,filed_date)

7) Flujo
1. Cargar CIKs.
2. Descargar companyfacts con retries/rate-limit.
3. Parsear facts a formato tabular.
4. Persistir raw + failures + manifest.

8) CLI
python -m data.edgar.fetch_companyfacts ^
  --cik-list-path data/edgar/mappings/ticker_cik_current.parquet ^
  --config-path configs/data/edgar_fetch_companyfacts.yaml ^
  --run-id fetch_companyfacts_20260305_001

9) DoD
- facts descargados y listos para parse_xbrl/point_in_time.

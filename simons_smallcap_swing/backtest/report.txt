\begin{abstract}
Este documento describe el diseño, la fundamentación matemática y el contexto académico/industrial del módulo \texttt{backtest/report.py} en el repositorio \texttt{JimSimons}. El objetivo es generar \textbf{reportes de backtest claros, auditables y accionables} a partir de los outputs del \texttt{engine} y, cuando aplica, de \texttt{walkforward} agregados. El módulo calcula métricas de performance neta, riesgo, costes y estabilidad, y produce tanto \textit{reporte ejecutivo} como \textit{reporte técnico} en formatos consumibles (JSON, \texttt{parquet}, HTML, Markdown). El diseño está inspirado en frameworks de reporte de \textit{quant trading} modernos que generan \textit{tearsheets} y HTML con muchas métricas y gráficos de equity curve, drawdown, turnover y costes \cite{quantstats, quantconnect, quantlite, quantshare}.
\end{abstract}

\section{Objetivo del módulo}

El módulo \texttt{backtest/report.py} tiene como objetivo:

\begin{itemize}
  \item \textbf{Consolidar} los outputs del backtest (P\&L, exposiciones, costes, \texttt{attribution}, \texttt{diagnostics}) en un informe unificado.
  \item \textbf{Producir métricas netas} validadas de rendimiento, riesgo y costes de transacción, totalmente alineadas con el motor de backtest.
  \item \textbf{Generar visualizaciones} interpretables de equity curve, drawdown, turnover y costes.
  \item \textbf{Incluir breakdown por subperiodo, lado (long/short), deciles de señal y sectores}, para apoyar decisiones de promoción y ajuste de estrategia.
\end{itemize}

El reporte debe ser suficientemente claro para:
\begin{itemize}
  \item \textbf{Resumen ejecutivo}: para PMs, partners y inversores.
  \item \textbf{Análisis técnico}: para equipos de investigación cuantitativa y \textit{risk management}.
\end{itemize}

\section{Contexto académico e industrial}

\subsection{Métricas de performance y riesgo}

El módulo calcula métricas estándar en \textit{quantitative finance} y \textit{backtesting} moderno:

\begin{itemize}
  \item \textbf{CAGR} (Compound Annual Growth Rate):
    \[
      \text{CAGR} = \left( \frac{\text{NAV}_T}{\text{NAV}_0} \right)^{1/T_y} - 1
    \]
    donde \(T_y\) es el número de años de backtest.

  \item \textbf{Volatilidad anualizada}:
    \[
      \text{Vol}_{\text{annual}} = \sigma_{\text{daily}} \sqrt{252}
    \]
    donde \(\sigma_{\text{daily}}\) es la desviación estándar de retornos diarios netos.

  \item \textbf{Sharpe ratio}:
    \[
      \text{Sharpe} = \frac{\mu_{\text{net}}}{\sigma_{\text{net}}}
    \]
    ratio de retorno anualizado sobre volatilidad anualizada.

  \item \textbf{Sortino ratio}:
    \[
      \text{Sortino} = \frac{\mu_{\text{net}}}{\sigma_{\text{down}}}
    \]
    volatilidad de rendimientos negativos, penalizando sólo el \textit{downside} \cite{sortino, riskmetrics}.

  \item \textbf{Calmar ratio}:
    \[
      \text{Calmar} = \frac{\text{CAGR}}{|\text{Max Drawdown}|}
    \]
    medición de \textit{risk‑adjusted return} vinculada a \textit{drawdown} extremo \cite{calmar, algostrategyanalyzer}.

  \item \textbf{Win rate y \textit{total cost in bps}}:
    \[
      \text{Win\_rate} = \frac{n_{\text{winning\_days}}}{T}
      \quad
      \text{Total\_cost\_bps} = \frac{1}{T} \sum_{t} \text{cost\_bps}_t
    \]
\end{itemize}

\subsection{Reportes de tipo tearsheet}

Frameworks como \texttt{QuantStats}, \texttt{QuantConnect} o \texttt{quantlite} generan \textit{tearsheets} HTML con:
\begin{itemize}
  \item \textit{Equity curves} y mapas de calor mensuales.
  \item \textit{Rolling Sharpe/Vol} y \textit{drawdown} top.
  \item Breakdown por deciles de \textit{alpha} y por sector \cite{quantstats, quantlite, quantconnect}.
\end{itemize}

El módulo \texttt{report.py} implementa una versión parametrizable y reproducible de este tipo de reporte, orientada a small caps y swing trading \cite{quantlite, quantconnect, quantshare}.

\section{Alcance funcional}

\subsection{Qué incluye el módulo}

\begin{itemize}
  \item \textbf{Cálculo de métricas agregadas} de retorno, riesgo y costes.
  \item \textbf{Graficación/tabulación de}:
    \begin{itemize}
      \item \textit{Equity curve} neto.
      \item \textit{Drawdown curve} y top drawdowns.
      \item \textit{Turnover} y \textit{trading activity}.
      \item \textit{Coste total} y \textit{bps por componente} (fees, slippage, impact, borrow).
    \end{itemize}
  \item \textbf{Breakdown por}:
    \begin{itemize}
      \item \textit{Subperiodo} (por año, trimestre o régimen).
      \item \textit{Lado} (long vs short).
      \item \textit{Deciles de señal} (para evaluar \textit{long–short spread} y calidad de \textit{alpha}).
      \item \textit{Sector} (si el motor lo proporciona).
    \end{itemize}
  \item \textbf{Export a formatos consumibles}:
    \begin{itemize}
      \item \texttt{JSON} (para \texttt{validation\_suite}).
      \item \texttt{parquet} (para investigación y \texttt{capacity\_analysis}).
      \item \texttt{Markdown} o \texttt{HTML} (para presentación interna del equipo).
    \end{itemize}
\end{itemize}

\subsection{Qué no incluye}

\begin{itemize}
  \item \textbf{Ejecución del backtest} (tarea de \texttt{backtest/engine.py}).
  \item \textbf{Calibración o \textit{tuning} del modelo}, que se realiza en \texttt{research} o \texttt{optimization}.
\end{itemize}

\section{Dependencias upstream y downstream}

\subsection{Entradas upstream}

\begin{description}
  \item[Backtest runs] \texttt{backtest/runs/*}:

      \item \texttt{positions\_daily.parquet}
      \item \texttt{pnl\_daily.parquet}

      \item \texttt{orders/fills} (opcional, para \textit{fill rate} y \textit{slippage}).
    \end{itemize}
  \item[Walkforward agregados] (si se usa \texttt{walkforward}).
  \item[Benchmark] Serie de retorno del benchmark (opcional, para \textit{relative performance}).
\end{description}

\subsection{Consumidores downstream}

\begin{itemize}
  \item \texttt{validation/validation\_suite.py}: validación de la calidad de la estrategia.
  \item \texttt{research/ablation/*}: análisis de \textit{ablation} y \textit{capacity\_analysis}.
  \item \textbf{Presentación interna} del equipo de inversión (PMs, \textit{risk}, \textit{ops}).
\end{itemize}

\section{Contrato de entrada y salida}

\subsection{Entradas mínimas}

\begin{itemize}
  \item \texttt{backtest\_run\_path}: ruta de \texttt{backtest/runs/\{run\_id\}/}.
  \item \texttt{report\_config\_path}: fichero YAML con:
    \begin{itemize}
      \item \texttt{sections\_enabled}
      \item \texttt{aggregation\_freq} (yearly, quarterly, by regime).
      \item \texttt{decile\_buckets}
      \item \texttt{benchmark\_relative}
    \end{itemize}
  \item \texttt{run\_id}: identificador del backtest.
\end{itemize}

\subsection{Entradas opcionales}

\begin{itemize}
  \item \texttt{benchmark\_path}: serie de retorno del benchmark.
  \item \texttt{factor\_returns\_path}: factores de riesgo (para \textit{risk attribution} opcional).
  \item \texttt{output\_format\_list}: lista de formatos de salida, p.ej. \texttt{[json, parquet, md, html]}.
\end{itemize}

\subsection{Salidas}

\begin{itemize}
  \item \texttt{backtest/reports/\{run\_id\}/report\_summary.json}
  \item \texttt{backtest/reports/\{run\_id\}/performance\_table.parquet}
  \item \texttt{backtest/reports/\{run\_id\}/risk\_table.parquet}
  \item \texttt{backtest/reports/\{run\_id\}/cost\_breakdown.parquet}
  \item \texttt{backtest/reports/\{run\_id\}/report.md} (o \texttt{.html})
  \item \texttt{backtest/reports/\{run\_id\}/manifest.json}
\end{itemize}

\section{Secciones recomendadas del reporte}

\subsection{Resumen ejecutivo}

\begin{itemize}
  \item \texttt{CAGR}, \texttt{Sharpe}, \texttt{Sortino}, \texttt{Calmar}.
  \item \texttt{Max drawdown} absoluto y relativo vs benchmark.
  \item \texttt{Turnover} promedio anual.
  \item \texttt{Total\_cost\_bps} y \textit{implementation shortfall}.
  \item \textit{Win rate} diario/mensual.
\end{itemize}

Estas métricas deben estar claramente desglosadas en \texttt{Net} vs Benchmark (si aplica).

\subsection{Performance detallada}

\begin{itemize}
  \item \textbf{Retornos diarios, mensuales y anuales}:
    \[
      r_{m} = \frac{1}{n_m} \sum_{t \in m} r_t
    \]
    con \textit{heatmap} mensual si se genera \texttt{HTML}.
  \item \textbf{Rolling Sharpe y volatilidad}:
    \[
      \text{Sharpe}^{(\tau)}_t = \frac{\mu^{(\tau)}_t}{\sigma^{(\tau)}_t}
    \]
  \item \textbf{Cumulative NAV} vs benchmark (equity curve).
\end{itemize}

\subsection{Riesgo}

\begin{itemize}
  \item \textbf{Drawdown curve}:
    \[
      \text{DD}_t = 1 - \frac{\text{NAV}_t}{\max_{s \leq t} \text{NAV}_s}
    \]
  \item \textbf{Top drawdowns} y \textit{duration} / \textit{recovery}.
  \item \textbf{Métricas de \textit{downside}}:
    \[
      \text{Sortino} = \frac{\mu}{\sigma_{\text{down}}}
    \]
  \item \textit{Time under water}:
    \[
      \text{TUW} = \frac{1}{T} \sum_{t} \mathbb{I}(\text{DD}_t > 0)
    \]
\end{itemize}

\subsection{Ejecución y costes}

\begin{itemize}
  \item \textbf{Implementation shortfall} por operación o por período:
    \[
      \text{IS}_t = \text{Modeled\_return}_t - \text{Executed\_return}_t
    \]
  \item \textbf{Coste total} y \textit{bps por componente}:
    \[
      C_{\text{total}} = C_{\text{fee}} + C_{\text{slippage}} + C_{\text{impact}} + C_{\text{borrow}}
    \]
  \item \textbf{Fill rate y participación media}:
    \[
      \text{Fill\_rate} = \frac{\sum_{i} \text{fill\_shares}_i}{\sum_{i} \text{target\_shares}_i}
      \quad
      \text{Participation} = \frac{\text{executed\_volume}}{\text{ADV}}
    \]
\end{itemize}

\subsection{Diagnóstico de señal}

\begin{itemize}
  \item \textbf{Long–short spread}:
    \[
      \text{LS\_spread} = \mu_{\text{top\_decile}} - \mu_{\text{bottom\_decile}}
    \]
  \item \textbf{Contribución por decil} de \textit{alpha}.
  \item \textbf{Estabilidad por subperiodo / régimen}:
    \[
      \text{Stability}_{\text{regime}} = \text{corr}( \text{signal}_t, \text{returns}_{t} ) \text{ condicionado al régimen}
    \]
\end{itemize}

\section{Métricas mínimas obligatorias}

El reporte debe incluir:

\begin{itemize}
  \item \texttt{total\_return}
  \item \texttt{CAGR}
  \item \texttt{annual\_vol}
  \item \texttt{Sharpe}
  \item \texttt{Sortino}
  \item \texttt{max\_drawdown}
  \item \texttt{Calmar}
  \item \texttt{win\_rate}
  \item \texttt{turnover\_avg}
  \item \texttt{total\_cost\_bps}
\end{itemize}

Estas métricas se guardan en \texttt{report\_summary.json} y \texttt{performance\_table.parquet} para consumo estandarizado por \texttt{validation} y \texttt{research}.

\section{Flujo del proceso}

\begin{enumerate}
  \item \textbf{Cargar outputs} del engine y, si aplica, de \texttt{walkforward}:
    \begin{itemize}
      \item \texttt{pnl\_daily}, \texttt{positions}, \texttt{exposures}, \texttt{cost\_breakdown}.
    \end{itemize}
  \item \textbf{Validar integridad de series}:
    \begin{itemize}
      \item Sin \texttt{NaN} críticos en \texttt{NAV} y retornos diarios.
      \item Alineación de \texttt{timestamps} y \texttt{run\_id}.
    \end{itemize}
  \item \textbf{Calcular métricas de performance/riesgo/coste}:
    \begin{itemize}
      \item \texttt{compute\_performance\_metrics(...)}
      \item \texttt{compute\_risk\_metrics(...)}
      \item \texttt{compute\_cost\_metrics(...)}
    \end{itemize}
  \item \textbf{Construir tablas y series derivadas}:
    \begin{itemize}
      \item \texttt{annual/quarterly/montly} returns.
      \item \texttt{rolling Sharpe/Vol}.
      \item \texttt{long/short/decile} breakdown.
    \end{itemize}
  \item \textbf{Renderizar reporte} en formatos solicitados:
    \begin{itemize}
      \item \texttt{Markdown} o \

Modulo: backtest/run_walkforward.py

1) Objetivo
Orquestar una evaluacion walk-forward end-to-end (train -> predict -> backtest) sobre ventanas temporales consecutivas, produciendo metricas OOS realistas, reproducibles y libres de leakage.

2) Alcance
Incluye:
- Definicion de ventanas rolling/expanding train-valid-test.
- Ejecucion secuencial de entrenamiento por ventana.
- Generacion de predicciones en bloque OOS.
- Lanzamiento del backtest engine por ventana.
- Agregacion global de resultados OOS.

No incluye:
- Implementacion interna de train_lgbm/predict/engine (los invoca).
- Tuneo hiperparametrico masivo.
- Seleccion de senales manual fuera de config.

3) Dependencias aguas arriba y aguas abajo
Upstream:
- features/store + feature_qc pass
- labels/store + purged_splits
- models/gbdt/train_lgbm
- models/inference/predict
- backtest/engine

Downstream:
- backtest/report.py
- validation/validation_suite.py
- research/ablation/*

4) Contrato de entrada
Entradas minimas:
- features_path
- labels_path
- prices_path
- universe_path
- walkforward_config_path
- run_id

Parametros clave de config:
- window_mode: rolling | expanding
- train_years
- valid_months
- test_months
- step_months
- min_train_days
- embargo_days
- model_config_path
- backtest_config_path
- seed

5) Contrato de salida
Por ventana:
- backtest/walkforward/{run_id}/window_{i}/model_artifacts/*
- backtest/walkforward/{run_id}/window_{i}/predictions.parquet
- backtest/walkforward/{run_id}/window_{i}/backtest_outputs/*
- backtest/walkforward/{run_id}/window_{i}/metrics.json

Agregado global:
- backtest/walkforward/{run_id}/oos_equity_curve.parquet
- backtest/walkforward/{run_id}/window_summary.parquet
- backtest/walkforward/{run_id}/global_metrics.json
- backtest/walkforward/{run_id}/manifest.json

6) Reglas PIT y anti-leakage
- Cada ventana entrena solo con datos anteriores al inicio de test.
- Validacion dentro de train window; test nunca se usa para tuning.
- Embargo entre train y test si esta configurado.
- Agregacion global usa unicamente segmentos OOS no solapados.

7) Flujo del proceso
1. Cargar config y construir calendario de ventanas.
2. Para cada ventana:
   - construir datasets train/valid/test.
   - ejecutar entrenamiento.
   - generar predicciones en test window.
   - ejecutar backtest engine sobre esa ventana OOS.
   - guardar outputs de ventana.
3. Concatenar resultados OOS de todas las ventanas.
4. Calcular metricas globales y por subperiodo.
5. Persistir reportes y manifest.

8) Metricas recomendadas
- CAGR, Sharpe, Sortino, max drawdown (global OOS)
- estabilidad por ventana (dispersion de Sharpe/IC)
- turnover y costes promedio
- hit ratio y spread long-short

9) Validaciones minimas
- ventanas sin solapamiento indebido en test.
- cada fecha OOS aparece una sola vez en agregado final.
- sin leakage flags en ventanas.
- tamano minimo de train/test por ventana.

10) Manejo de fallos
- Si falla una ventana: marcar fail_window y continuar segun politica.
- Politica configurable:
  - fail_fast=true: aborta run.
  - fail_fast=false: continua y registra excepcion.

11) Observabilidad
Logging:
- INFO: progreso por ventana.
- WARNING: ventana con datos escasos/metricas inestables.
- ERROR: fallo de entrenamiento o backtest en ventana.

Metricas operativas:
- n_windows_total
- n_windows_success
- n_windows_failed
- total_runtime_sec

12) CLI sugerida
python -m backtest.run_walkforward ^
  --features-path features/store ^
  --labels-path labels/store ^
  --prices-path data/price/adjusted/mode=dual_output ^
  --universe-path data/universe/history ^
  --config-path configs/backtest/walkforward.yaml ^
  --run-id walkforward_20260305_001

13) Estructura interna recomendada
- load_walkforward_config(...)
- build_time_windows(calendar, cfg)
- run_window_training(window, paths, cfg)
- run_window_inference(window, model_artifacts, paths, cfg)
- run_window_backtest(window, preds_path, paths, cfg)
- aggregate_oos_results(window_outputs)
- compute_global_metrics(df_oos)
- persist_walkforward_outputs(...)
- main()

14) Criterios de aceptacion
- Pipeline completo ejecuta por ventanas y agrega OOS sin leakage.
- Outputs por ventana y globales reproducibles.
- Metricas globales y de estabilidad disponibles.

15) Casos de prueba minimos
- rolling windows bien construidas y sin solape de test.
- expanding windows respetan crecimiento de train.
- fallo en una ventana se maneja segun fail_fast.
- agregado OOS no duplica fechas.

16) Checklist rapido
- [ ] Constructor de ventanas robusto.
- [ ] Orquestacion train->predict->backtest.
- [ ] Agregacion OOS limpia.
- [ ] Reportes por ventana y global.
- [ ] Manifest y logs completos.

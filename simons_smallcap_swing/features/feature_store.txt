Modulo: features/feature_store.py

1) Objetivo
Persistir y servir la matriz de features PIT con versionado, particionado y contratos de lectura estables para training/inference.

2) Alcance
Incluye escritura, lectura, catalogo de versiones, schema enforcement y snapshots reproducibles.

3) Entradas
- features_df (date,symbol,feature_cols)
- feature_dictionary
- store_config_path
- run_id

4) Salidas
- features/store/date=YYYY-MM-DD/part-*.parquet
- features/store/_meta/feature_dictionary_{run_id}.json
- features/store/_meta/store_manifest_{run_id}.json
- features/store/_meta/schema_{run_id}.json

5) Reglas
- clave primaria logica: (date,symbol)
- schema versionado y validado al escribir
- lectura por date range + column projection
- snapshots inmutables por run

6) Operaciones core
- write_features_snapshot(...)
- read_features_range(...)
- read_latest_snapshot(...)
- validate_store_schema(...)
- list_store_versions(...)

7) Validaciones
- no duplicados PK
- columnas requeridas presentes
- tipos numericos coherentes
- sin corrupcion de particiones

8) Observabilidad
- n_rows_written
- n_dates_written
- n_features
- write_read_latency_sec

9) CLI
python -m features.feature_store ^
  --input-path features/tmp/build_features_20260305_001.parquet ^
  --config-path configs/features/store.yaml ^
  --run-id feature_store_20260305_001

10) DoD
- store estable y consultable por training/inference
- metadata completa de versionado

Modulo: features/fundamentals_core.py

1) Objetivo
Construir features fundamentales base (value/quality/profitability/leverage/growth) desde dataset EDGAR PIT normalizado.

2) Alcance
Incluye ratios core por simbolo/fecha con reglas de disponibilidad PIT y manejo de staleness.

3) Entradas
- fundamentals_pit_path
- prices_path (para ratios de market value)
- fundamentals_core_config_path
- run_id

4) Salidas
- features/fundamentals_core/date={yyyy-mm-dd}/part-*.parquet
- features/fundamentals_core/summary_{run_id}.json
- features/fundamentals_core/manifest_{run_id}.json

5) Features core sugeridas
- fund_pe_proxy / earnings_yield
- fund_book_to_market
- fund_profit_margin
- fund_roa / fund_roe_proxy
- fund_leverage_ratio
- fund_asset_turnover

6) Reglas PIT
- usar metricas disponibles por asof_date.
- forward-fill limitado por max_staleness por metrica.
- excluir observaciones stale severas segun policy.

7) Flujo
1. Cargar PIT fundamentals y precios.
2. Calcular ratios base.
3. Aplicar controles de staleness y outliers.
4. Persistir outputs.

8) Validaciones
- denominadores no cero o fallback.
- ratios en rangos plausibles.
- cobertura minima por metrica.

9) CLI
python -m features.fundamentals_core ^
  --fundamentals-pit-path data/edgar/pit ^
  --prices-path data/price/adjusted/mode=dual_output ^
  --config-path configs/features/fundamentals_core.yaml ^
  --run-id fundamentals_core_20260305_001

10) DoD
- set fundamental core PIT listo para modelado.
